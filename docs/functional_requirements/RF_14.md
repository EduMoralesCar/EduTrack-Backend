# Requerimiento Funcional: RF 14 - Detección de riesgo académico (Plantilla Educativa)

## Explicación del Requerimiento (Genérico)
Este requerimiento define el flujo de comunicación estándar para la **eliminación o desestimación** (lógica o física) de una alerta de riesgo académico en el sistema. Permite que un usuario administrador o consejero elimine una alerta que ya ha sido atendida o que es un falso positivo, actualizando o removiendo el registro en PostgreSQL.

---

## Estructura del Flujo de Comunicación (Plantilla educativa)

### 1. Frontend (React)
El flujo comienza en el componente React `[ComponenteReact]`, cuando un usuario presiona un botón para descartar o eliminar una alerta de riesgo académico, solicitando previa confirmación.

```javascript
// [ComponenteReact].jsx
import React from 'react';

export const ComponenteRiesgo = ({ alertas, setAlertas }) => {
  const handleEliminarAlerta = async (id) => {
    const confirmar = window.confirm("¿Está seguro de que desea descartar esta alerta?");
    if (!confirmar) return;

    try {
      await fetchEliminarAlerta(id);
      // Actualiza el estado local filtrando la alerta eliminada
      setAlertas(alertas.filter(alerta => alerta.id !== id));
      console.log('Alerta eliminada exitosamente');
    } catch (error) {
      console.error('Error al descartar la alerta:', error);
    }
  };

  return (
    <ul>
      {alertas.map(alerta => (
        <li key={alerta.id}>
          Alerta #{alerta.id}: {alerta.descripcion}
          <button onClick={() => handleEliminarAlerta(alerta.id)}>Descartar</button>
        </li>
      ))}
    </ul>
  );
};
```

### 2. Red (HTTP Call)
La llamada de red se realiza mediante una petición HTTP DELETE hacia la API del backend.

- **Método HTTP:** `DELETE`
- **Ruta/URL:** `[EndpointUrl]/{id}` (Ejemplo: `/api/v1/alertas-riesgo/123`)
- **Cabeceras (Headers):** `Accept: application/json`
- **Cuerpo (Request Body):** Vacío.

### 3. Controller (Spring Boot)
El controlador `[ControllerClass]` recibe el identificador en la variable de ruta (`@PathVariable`) y delega la eliminación a la capa de servicio. Si es exitoso, responde con un código de estado `204 No Content`.

```java
// [ControllerClass].java
@RestController
@RequestMapping("/api/v1/alertas-riesgo")
public class ControllerClass {

    private final ServiceClass serviceClass;

    public ControllerClass(ServiceClass serviceClass) {
        this.serviceClass = serviceClass;
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> eliminarAlerta(@PathVariable Long id) {
        serviceClass.eliminarAlerta(id);
        return ResponseEntity.noContent().build(); // Devuelve estado HTTP 204
    }
}
```

### 4. Service
El servicio `[ServiceClass]` valida la existencia del registro y realiza la eliminación física (`deleteById`) o la desactivación lógica (cambiando una bandera en la entidad).

```java
// [ServiceClass].java
@Service
public class ServiceClass {

    private final RepositoryInterface repositoryInterface;

    public ServiceClass(RepositoryInterface repositoryInterface) {
        this.repositoryInterface = repositoryInterface;
    }

    @Transactional
    public void eliminarAlerta(Long id) {
        if (!repositoryInterface.existsById(id)) {
            throw new ResourceNotFoundException("Alerta de riesgo no encontrada con ID: " + id);
        }
        
        // Opción A: Eliminación Física
        repositoryInterface.deleteById(id);

        /*
        // Opción B: Eliminación Lógica (cambiar bandera 'activo' a false)
        JPAEntity alerta = repositoryInterface.findById(id).orElseThrow();
        alerta.setActivo(false);
        repositoryInterface.save(alerta);
        */
    }
}
```

### 5. Repository & Entity
La interfaz `[RepositoryInterface]` interactúa con la base de datos PostgreSQL. La entidad `[JPAEntity]` mapea los campos correspondientes de la tabla.

```java
// [RepositoryInterface].java
@Repository
public interface RepositoryInterface extends JpaRepository<JPAEntity, Long> {
}

// [JPAEntity].java
@Entity
@Table(name = "tabla_alertas_riesgo")
public class JPAEntity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "descripcion", nullable = false)
    private String descripcion;

    @Column(name = "activo")
    private boolean activo = true;

    // Getters y Setters
}
```

### 6. Return Path
El flujo de retorno es el siguiente:
1. **Base de Datos:** Ejecuta la instrucción SQL `DELETE` o `UPDATE` en PostgreSQL y confirma la transacción.
2. **Repository:** Informa la confirmación de la persistencia a `[ServiceClass]`.
3. **Service:** Finaliza la transacción de manera limpia.
4. **Controller:** Retorna un objeto `ResponseEntity` con estado `204 No Content` sin cuerpo de respuesta.
5. **Red:** Envía la respuesta HTTP al cliente frontend.
6. **Frontend (React):** Recibe la confirmación HTTP 204, actualiza el estado local removiendo el elemento de la lista y redibuja la interfaz.
