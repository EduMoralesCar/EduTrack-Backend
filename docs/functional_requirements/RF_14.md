# Requerimiento Funcional: [RF_14 - Nombre del Requerimiento de Eliminación] (Plantilla Educativa)

## Explicación del Requerimiento (Genérico)
Esta plantilla describe el flujo de comunicación estándar para la **eliminación** (lógica o física) de un recurso en el sistema. Se ejemplifica tanto la eliminación física (eliminar el registro de la base de datos) como la eliminación lógica (cambiar un estado/campo bandera `activo` a falso) para evitar la pérdida permanente de información histórica.

---

## Estructura del Flujo de Comunicación (Plantilla educativa)

### 1. Frontend (React)
El flujo comienza en el componente React `[ComponenteReactLista]` o `[ComponenteReactDetalle]` al presionar un botón de eliminar (previa confirmación por interfaz).

```javascript
// [ComponenteReactLista].jsx
import React from 'react';

export const ComponenteLista = ({ recursos, setRecursos }) => {
  const handleEliminar = async (id) => {
    const confirmar = window.confirm("¿Está seguro de que desea eliminar este recurso?");
    if (!confirmar) return;

    try {
      await fetchEliminarRecurso(id);
      // Remueve el recurso de la lista en la UI sin recargar
      setRecursos(recursos.filter(item => item.id !== id));
      console.log('Recurso eliminado con éxito');
    } catch (error) {
      console.error('Error al eliminar recurso:', error);
    }
  };

  return (
    <ul>
      {recursos.map(item => (
        <li key={item.id}>
          {item.campoUno} 
          <button onClick={() => handleEliminar(item.id)}>Eliminar</button>
        </li>
      ))}
    </ul>
  );
};
```

### 2. Red (HTTP Call)
La llamada de red se realiza mediante una petición HTTP DELETE.

- **Método HTTP:** `DELETE`
- **Ruta/URL:** `[EndpointUrl]/{id}` (Ejemplo: `/api/v1/recursos/123`)
- **Cabeceras (Headers):** `Accept: application/json`
- **Cuerpo (Request Body):** Vacío.

### 3. Controller (Spring Boot)
El controlador `[ControllerClass]` recibe la solicitud DELETE y llama al servicio. Devuelve un estado `204 No Content` si la eliminación fue exitosa.

```java
// [ControllerClass].java
@RestController
@RequestMapping("/api/v1/recursos")
public class ControllerClass {

    private final ServiceClass serviceClass;

    public ControllerClass(ServiceClass serviceClass) {
        this.serviceClass = serviceClass;
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> eliminarRecurso(@PathVariable Long id) {
        serviceClass.eliminarRecurso(id);
        return ResponseEntity.noContent().build(); // Estado HTTP 204
    }
}
```

### 4. Service
El servicio `[ServiceClass]` contiene la lógica para la eliminación física (`deleteById`) o eliminación lógica (modificar la entidad).

```java
// [ServiceClass].java
@Service
public class ServiceClass {

    private final RepositoryInterface repositoryInterface;

    public ServiceClass(RepositoryInterface repositoryInterface) {
        this.repositoryInterface = repositoryInterface;
    }

    @Transactional
    public void eliminarRecurso(Long id) {
        // Opción A: Eliminación Física
        if (!repositoryInterface.existsById(id)) {
            throw new ResourceNotFoundException("Recurso no encontrado con ID: " + id);
        }
        repositoryInterface.deleteById(id);

        /* 
        // Opción B: Eliminación Lógica (Recomendado para resguardar integridad de datos)
        JPAEntity recurso = repositoryInterface.findById(id)
                .orElseThrow(() -> new ResourceNotFoundException("Recurso no encontrado"));
        recurso.setActivo(false);
        repositoryInterface.save(recurso);
        */
    }
}
```

### 5. Repository & Entity
La interfaz `[RepositoryInterface]` interactúa directamente con PostgreSQL.

```java
// [RepositoryInterface].java
@Repository
public interface RepositoryInterface extends JpaRepository<JPAEntity, Long> {
}

// [JPAEntity].java
@Entity
@Table(name = "tabla_recursos")
public class JPAEntity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "campo_uno")
    private String campoUno;

    @Column(name = "activo")
    private boolean activo = true; // Para la opción de eliminación lógica
}
```

### 6. Return Path
El flujo de retorno es el siguiente:
1. **Base de Datos:** Ejecuta la sentencia SQL (`DELETE FROM tabla_recursos WHERE id = ?` o `UPDATE ... SET activo = false ...`).
2. **Repository:** Confirma la ejecución al Service.
3. **Service:** Finaliza la transacción de forma exitosa.
4. **Controller:** Construye el `ResponseEntity.noContent()` sin cuerpo de respuesta.
5. **Red:** Envía la respuesta HTTP con estado `204 No Content` al cliente frontend.
6. **Frontend (React):** Resuelve la promesa de fetch/axios exitosamente y actualiza el estado de los componentes locales para refrescar la lista.
