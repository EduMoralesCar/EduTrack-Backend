# Requerimiento Funcional: RF 11 - Cálculo automático de promedios (Plantilla Educativa)

## Explicación del Requerimiento (Genérico)
Este requerimiento define el flujo para realizar el cálculo automático de promedios en el sistema. Describe cómo el frontend envía los datos de las calificaciones o solicita el cálculo para un conjunto de registros, y cómo el backend procesa esta información de manera automática, actualiza los registros correspondientes en la base de datos PostgreSQL y retorna los resultados calculados.

---

## Estructura del Flujo de Comunicación (Plantilla educativa)

### 1. Frontend (React)
El flujo comienza en el componente React `[ComponenteReact]`, el cual recopila los datos necesarios para el cálculo (por ejemplo, ID del estudiante y notas) y los envía al backend al presionar un botón.

```javascript
// [ComponenteReact].jsx
import React, { useState } from 'react';

export const ComponenteCalculo = () => {
  const [datos, setDatos] = useState({
    estudianteId: '',
    calificaciones: []
  });

  const handleCalcular = async (e) => {
    e.preventDefault();
    try {
      // Invocación al servicio de red
      const resultado = await fetchCalcularPromedio(datos);
      console.log('Cálculo realizado con éxito:', resultado);
    } catch (error) {
      console.error('Error al calcular el promedio:', error);
    }
  };

  return (
    <form onSubmit={handleCalcular}>
      {/* Inputs para ingresar datos */}
      <button type="submit">Calcular Promedio</button>
    </form>
  );
};
```

### 2. Red (HTTP Call)
La llamada de red se realiza mediante una petición HTTP POST enviando la información estructurada en JSON.

- **Método HTTP:** `POST`
- **Ruta/URL:** `[EndpointUrl]` (Ejemplo: `/api/v1/promedios/calcular`)
- **Cabeceras (Headers):** `Content-Type: application/json`
- **Cuerpo (Request Body):**
```json
{
  "estudianteId": 123,
  "calificaciones": [15.5, 18.0, 14.2]
}
```

### 3. Controller (Spring Boot)
El controlador `[ControllerClass]` recibe la solicitud HTTP POST, valida el cuerpo del mensaje y delega la ejecución de la lógica al servicio correspondiente.

```java
// [ControllerClass].java
@RestController
@RequestMapping("/api/v1/promedios")
public class ControllerClass {

    private final ServiceClass serviceClass;

    public ControllerClass(ServiceClass serviceClass) {
        this.serviceClass = serviceClass;
    }

    @PostMapping("/calcular")
    public ResponseEntity<JPAEntity> calcularPromedio(@Valid @RequestBody DTORequest dto) {
        JPAEntity resultado = serviceClass.procesarCalculo(dto);
        return new ResponseEntity<>(resultado, HttpStatus.CREATED);
    }
}
```

### 4. Service
El servicio `[ServiceClass]` contiene la lógica de negocio para realizar la operación aritmética del promedio, actualizar el estado de la entidad y persistir los cambios.

```java
// [ServiceClass].java
@Service
public class ServiceClass {

    private final RepositoryInterface repositoryInterface;

    public ServiceClass(RepositoryInterface repositoryInterface) {
        this.repositoryInterface = repositoryInterface;
    }

    @Transactional
    public JPAEntity procesarCalculo(DTORequest dto) {
        // Lógica de negocio: Calcular el promedio a partir de la lista de calificaciones
        double suma = 0;
        for (double nota : dto.getCalificaciones()) {
            suma += nota;
        }
        double promedio = dto.getCalificaciones().isEmpty() ? 0.0 : suma / dto.getCalificaciones().size();

        // Mapear y guardar en base de datos
        JPAEntity entidad = new JPAEntity();
        entidad.setEstudianteId(dto.getEstudianteId());
        entidad.setPromedio(promedio);
        entidad.setEstado(promedio >= 10.5 ? "Aprobado" : "Reprobado");

        return repositoryInterface.save(entidad);
    }
}
```

### 5. Repository & Entity
El repositorio `[RepositoryInterface]` realiza la persistencia en PostgreSQL mediante Spring Data JPA. La entidad `[JPAEntity]` define los campos de la tabla.

```java
// [RepositoryInterface].java
@Repository
public interface RepositoryInterface extends JpaRepository<JPAEntity, Long> {
}

// [JPAEntity].java
@Entity
@Table(name = "tabla_promedios")
public class JPAEntity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "estudiante_id", nullable = false)
    private Long estudianteId;

    @Column(name = "promedio", nullable = false)
    private Double promedio;

    @Column(name = "estado")
    private String estado;

    // Getters y Setters
}
```

### 6. Return Path
El flujo de retorno es el siguiente:
1. **Base de Datos:** Guarda el registro del promedio calculado y retorna el ID autogenerado a `[RepositoryInterface]`.
2. **Repository:** Devuelve la entidad guardada a `[ServiceClass]`.
3. **Service:** Finaliza la transacción y retorna la entidad a `[ControllerClass]`.
4. **Controller:** Construye el `ResponseEntity` con el estado `201 Created` y la entidad en el cuerpo JSON, y lo envía de regreso.
5. **Red:** Envía la respuesta HTTP con el JSON al cliente.
6. **Frontend (React):** `fetchCalcularPromedio` resuelve la promesa, lo que permite al componente `[ComponenteReact]` actualizar la interfaz mostrando el promedio calculado y el estado del estudiante.
