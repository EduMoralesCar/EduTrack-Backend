# RF 02: Autenticación de usuarios

- **Explicación del Requerimiento**:
  Este requerimiento permite a los usuarios registrados iniciar sesión en la plataforma utilizando sus credenciales (nombre de usuario y contraseña). Si la autenticación es exitosa, el sistema genera un token JWT (JSON Web Token) que se asocia al rol del usuario (`ADMIN`, `TEACHER`, `STUDENT`) para autorizar llamadas posteriores a los endpoints protegidos.

- **Flujo de Código Paso a Paso**:

  ### 1. Interfaz de Usuario (Frontend)
  El componente React responsable de la vista de login es [Login.jsx](file:///d:/Workspace/utp/EduTrack/Frontend/src/components/Login.jsx). Al enviar el formulario (evento `onSubmit`), se ejecuta la función `handleSubmit`:

  ```javascript
  const handleSubmit = async (e) => {
    e.preventDefault();
    if (!username || !password) {
      setError('Por favor, complete todos los campos');
      return;
    }

    setError('');
    setLoading(true);

    try {
      const data = await api.auth.login(username, password);
      onLoginSuccess({
        id: data.userId,
        username: data.username || username,
        role: data.role
      });
    } catch (err) {
      setError(err.message || 'Error de inicio de sesión. Verifique sus credenciales.');
    } finally {
      setLoading(false);
    }
  };
  ```

  La llamada a la API utiliza el módulo encapsulado en [api.js](file:///d:/Workspace/utp/EduTrack/Frontend/src/services/api.js):

  ```javascript
  export const api = {
    auth: {
      login: async (username, password) => {
        const data = await request('/auth/login', {
          method: 'POST',
          body: JSON.stringify({ username, password }),
        });
        if (data && data.token) {
          localStorage.setItem('edutrack_token', data.token);
          localStorage.setItem('edutrack_user', JSON.stringify({
            id: data.userId,
            username: data.username || username,
            role: data.role
          }));
        }
        return data;
      },
      // ...
    }
  };
  ```

  ### 2. Capa de Red
  - **Método HTTP**: `POST`
  - **URL Base**: `http://localhost:8081/api`
  - **Ruta (Path)**: `/auth/login`
  - **Parámetros**: Ninguno.
  - **Cuerpo de la Petición (Request Body)**: Formato JSON.
    ```json
    {
      "username": "JUAN PABLO",
      "password": "Admin123!"
    }
    ```

  ### 3. Controlador (Backend Controller)
  El controlador Spring Boot encargado de recibir la solicitud es [AuthController.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/web/controller/AuthController.java):

  ```java
  @RestController
  @RequestMapping("/api/auth")
  @RequiredArgsConstructor
  @Tag(name = "Autenticación", description = "RF02 - Login y sesión actual")
  public class AuthController {

      private final AuthService authService;

      @PostMapping("/login")
      @Operation(summary = "Iniciar sesión", description = "No requiere Authorize. Devuelve JWT según el rol.")
      public ResponseEntity<LoginResponseDTO> login(@Valid @RequestBody LoginRequestDTO request) {
          return ResponseEntity.ok(authService.login(request));
      }
  }
  ```

  ### 4. Capa de Negocio (Backend Service)
  La lógica de autenticación se gestiona en [AuthService.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/domain/service/AuthService.java). Delegará la autenticación en el `AuthenticationManager` de Spring Security, el cual internamente llama a `loadUserByUsername` de `CustomUserDetailsService` y verifica la contraseña contra el encriptador BCrypt:

  ```java
  public LoginResponseDTO login(LoginRequestDTO request) {
      Authentication authentication = authenticationManager.authenticate(
              new UsernamePasswordAuthenticationToken(request.getUsername(), request.getPassword())
      );

      UserPrincipal principal = (UserPrincipal) authentication.getPrincipal();
      String token = jwtService.generateToken(principal);

      return LoginResponseDTO.builder()
              .token(token)
              .tokenType("Bearer")
              .userId(principal.getId())
              .username(principal.getUsername())
              .role(principal.getRole())
              .build();
  }
  ```

  ### 5. Capa de Persistencia (Repository & Entity)
  - **Repositorio**: [UserRepository.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/persistance/repository/UserRepository.java). Durante el flujo de autenticación, el `CustomUserDetailsService` invoca al repositorio:
    ```java
    @Service
    @RequiredArgsConstructor
    public class CustomUserDetailsService implements UserDetailsService {
        private final UserRepository userRepository;

        @Override
        public UserDetails loadUserByUsername(String username) {
            return userRepository.findByUsernameIgnoreCase(username)
                    .map(UserPrincipal::new)
                    .orElseThrow(() -> new UsernameNotFoundException("Usuario no encontrado: " + username));
        }
    }
    ```
  - **Entidad**: [User.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/persistance/entity/User.java). La consulta a la tabla `users` se realiza filtrando por el nombre de usuario de manera insensible a mayúsculas y minúsculas:
    `SELECT * FROM users WHERE LOWER(username) = LOWER(?)`
    Luego Spring Security compara el hash de la contraseña en base de datos (`password`) con el string provisto por el usuario.

  ### 6. Camino de Retorno
  Una vez autenticado exitosamente:
  1. `AuthService` genera el JWT y construye el `LoginResponseDTO`.
  2. El controlador responde con estado `200 OK` y el DTO serializado en formato JSON.
  3. En frontend, la función wrapper de `api.js` guarda el `token` (bajo la clave `edutrack_token`) y el objeto simplificado del usuario (bajo la clave `edutrack_user`) en el `localStorage` del navegador.
  4. En `Login.jsx`, la promesa se resuelve con éxito y llama al callback `onLoginSuccess({ id, username, role })`.
  5. Este callback (definido en `App.jsx`) actualiza el estado principal de la aplicación (`setUser`), lo que provoca que React desmonte el componente de login y monte la interfaz del dashboard correspondiente a los privilegios del rol autenticado.
