# RF 07: Entrega de evaluaciones

- **Explicación del Requerimiento (Genérico)**:
  Este requerimiento define el proceso mediante el cual los usuarios (por ejemplo, estudiantes) suben o envían sus respuestas o evidencias de desarrollo de una evaluación. El sistema debe validar que la entrega se realice dentro del límite de intentos permitidos y antes de la fecha de vencimiento. Generalmente, implica subir un archivo físico (PDF, zip, etc.) o ingresar un texto de entrega, almacenando los metadatos en la base de datos PostgreSQL y opcionalmente el archivo físico en un almacenamiento de archivos (Local o Cloud Storage).

- **Estructura del Flujo de Comunicación (Plantilla educativa)**:

  ### 1. Frontend (React)
  El componente React `[ComponenteReact]` expone una interfaz que permite cargar archivos (usando un input de tipo `file`) o introducir texto libre. Al accionar el envío, se empaqueta la información en un formato adecuado para envío de binarios (`FormData`) y se despacha al backend.

  ```javascript
  // [ComponenteReact].jsx
  import React, { useState } from 'react';

  const [ComponenteReact] = ({ evaluacionId }) => {
    const [archivo, setArchivo] = useState(null);
    const [comentario, setComentario] = useState('');

    const handleFileChange = (e) => {
      setArchivo(e.target.files[0]);
    };

    const handleUpload = async (e) => {
      e.preventDefault();
      if (!archivo) return alert('Por favor, selecciona un archivo');

      // Se usa FormData para enviar archivos binarios por HTTP
      const formData = new FormData();
      formData.append('archivo', archivo);
      formData.append('comentario', comentario);
      formData.append('evaluacionId', evaluacionId);

      try {
        const response = await fetch('[EndpointUrl]', {
          method: 'POST',
          headers: {
            // Nota: Con FormData no se debe definir manualmente el Content-Type,
            // el navegador establecerá automáticamente el boundary.
            'Authorization': 'Bearer [TokenDeAutenticacion]'
          },
          body: formData
        });

        if (!response.ok) {
          throw new Error('Error al enviar la entrega');
        }

        const data = await response.json();
        alert('Entrega enviada con éxito');
      } catch (err) {
        console.error('Error al subir archivo:', err);
      }
    };

    return (
      <form onSubmit={handleUpload}>
        <input type="file" onChange={handleFileChange} />
        <textarea value={comentario} onChange={(e) => setComentario(e.target.value)} />
        <button type="submit">Enviar Entrega</button>
      </form>
    );
  };

  export default [ComponenteReact];
  ```

  ### 2. Red (HTTP Call)
  La comunicación de red transporta un formato multipart para adjuntos:
  - **Método HTTP**: `POST`
  - **Ruta de la API**: `[EndpointUrl]` (Ejemplo: `/api/v1/entregas`)
  - **Encabezados (Headers)**:
    - `Content-Type: multipart/form-data; boundary=----WebKitFormBoundary...`
    - `Authorization: Bearer [TokenDeAutenticacion]`
  - **Cuerpo del Mensaje (Request Body)**: Contenido estructurado en múltiples partes (boundary) conteniendo el archivo binario y los campos de metadatos asociados.

  ### 3. Controller (Spring Boot)
  La clase controladora `[ControllerClass]` recibe el payload multipart en Spring Boot mediante `@RequestParam` y anotaciones para el archivo adjunto (`MultipartFile`).

  ```java
  // [ControllerClass].java
  package [PackageController];

  import org.springframework.http.HttpStatus;
  import org.springframework.http.ResponseEntity;
  import org.springframework.web.bind.annotation.*;
  import org.springframework.web.multipart.MultipartFile;

  @RestController
  @RequestMapping("[EndpointUrl]")
  public class [ControllerClass] {

      private final [ServiceClass] service;

      public [ControllerClass]([ServiceClass] service) {
          this.service = service;
      }

      @PostMapping(consumes = "multipart/form-data")
      public ResponseEntity<[DTOClass]> uploadEntrega(
              @RequestParam("archivo") MultipartFile file,
              @RequestParam("comentario") String comentario,
              @RequestParam("evaluacionId") Long evaluacionId) {
          
          [DTOClass] result = service.guardarEntrega(file, comentario, evaluacionId);
          return ResponseEntity.status(HttpStatus.CREATED).body(result);
      }
  }
  ```

  ### 4. Service
  La clase de servicio `[ServiceClass]` contiene la lógica de negocio. Realiza las siguientes tareas:
  1. Validar si la evaluación existe en el sistema y si la fecha límite no ha expirado.
  2. Consultar si el estudiante no ha excedido la cantidad máxima de intentos permitidos para dicha evaluación.
  3. Guardar el archivo en el sistema de almacenamiento físico (ej. localmente en una carpeta predefinida o en Cloud Storage) y obtener su URL o ruta de acceso.
  4. Mapear e insertar el registro de la entrega en la base de datos PostgreSQL.

  ```java
  // [ServiceClass].java
  package [PackageService];

  import org.springframework.stereotype.Service;
  import org.springframework.transaction.annotation.Transactional;
  import org.springframework.web.multipart.MultipartFile;
  import java.io.IOException;

  @Service
  public class [ServiceClass] {

      private final [RepositoryInterface] repository;
      private final [FileStorageService] fileStorage;

      public [ServiceClass]([RepositoryInterface] repository, [FileStorageService] fileStorage) {
          this.repository = repository;
          this.fileStorage = fileStorage;
      }

      @Transactional
      public [DTOClass] guardarEntrega(MultipartFile file, String comentario, Long evaluacionId) {
          // 1. Validar negocio (ej: comprobar intentos previos)
          int intentos = repository.countByEvaluacionIdAndUsuarioId(evaluacionId, [IdUsuarioActual]);
          if (intentos >= [MaxIntentosPermitidos]) {
              throw new IllegalStateException("Límite de intentos alcanzado");
          }

          // 2. Almacenar el archivo físico
          String rutaArchivo;
          try {
              rutaArchivo = fileStorage.store(file);
          } catch (IOException e) {
              throw new RuntimeException("Error guardando el archivo físico", e);
          }

          // 3. Crear entidad y persistir
          [JPAEntity] entity = new [JPAEntity]();
          entity.setEvaluacionId(evaluacionId);
          entity.setUsuarioId([IdUsuarioActual]);
          entity.setComentario(comentario);
          entity.setRutaArchivo(rutaArchivo);
          entity.setFechaEntrega(java.time.LocalDateTime.now());

          [JPAEntity] saved = repository.save(entity);

          // 4. Retornar DTO de respuesta
          return mapToDto(saved);
      }

      private [DTOClass] mapToDto([JPAEntity] entity) {
          [DTOClass] dto = new [DTOClass]();
          dto.setId(entity.getId());
          dto.setEvaluacionId(entity.getEvaluacionId());
          dto.setComentario(entity.getComentario());
          dto.setRutaArchivo(entity.getRutaArchivo());
          dto.setFechaEntrega(entity.getFechaEntrega());
          return dto;
      }
  }
  ```

  ### 5. Repository & Entity
  - **Repository**: La interfaz `[RepositoryInterface]` declara métodos específicos de consulta (como `countByEvaluacionIdAndUsuarioId`).
  - **Entity**: La clase `[JPAEntity]` mapea los campos de metadatos de la entrega en la base de datos PostgreSQL.

  ```java
  // [RepositoryInterface].java
  package [PackageRepository];

  import org.springframework.data.jpa.repository.JpaRepository;
  import org.springframework.stereotype.Repository;

  @Repository
  public interface [RepositoryInterface] extends JpaRepository<[JPAEntity], Long> {
      int countByEvaluacionIdAndUsuarioId(Long evaluacionId, Long usuarioId);
  }
  ```

  ```java
  // [JPAEntity].java
  package [PackageEntity];

  import jakarta.persistence.*;
  import java.time.LocalDateTime;

  @Entity
  @Table(name = "[NombreTablaEntrega]")
  public class [JPAEntity] {

      private Long evaluacionId;

      @Column(name = "usuario_id", nullable = false)
      private Long usuarioId;

      @Column(length = 250)
      private String comentario;

      @Column(name = "ruta_archivo", nullable = false)

      @Column(name = "fecha_entrega", nullable = false)
      private LocalDateTime fechaEntrega;

      // Getters, Setters y Constructores
  }
  ```

  ### 6. Return Path
  1. **PostgreSQL**: Graba la tupla de entrega y devuelve el registro con su ID asignado en base de datos.
  2. **[RepositoryInterface]**: Retorna el objeto `[JPAEntity]` persistido.
  3. **[ServiceClass]**: Convierte la entidad a `[DTOClass]` y la retorna de vuelta al controlador.
  4. **[ControllerClass]**: Encapsula el `[DTOClass]` en un `ResponseEntity` de Spring Boot con estado `201 Created` y lo envía en formato JSON por HTTP.
  5. **Red**: Transmite la respuesta HTTP con el cuerpo JSON al navegador del cliente.
  6. **[ComponenteReact]**: Recibe el estado de éxito, actualiza la UI mostrando un mensaje confirmando el envío y puede deshabilitar el botón de entrega o actualizar la lista de entregas realizadas.
