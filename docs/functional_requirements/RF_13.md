# Requerimiento Funcional: [RF_13 - Nombre del Requerimiento de Actualización] (Plantilla Educativa)

## Explicación del Requerimiento (Genérico)
Esta plantilla describe el flujo de comunicación estándar para la **actualización** de un recurso existente en el sistema. El objetivo es permitir que un usuario edite los datos precargados en un formulario y guarde los cambios en la base de datos PostgreSQL, validando la existencia previa del registro.

---

## Estructura del Flujo de Comunicación (Plantilla educativa)

### 1. Frontend (React)
El flujo comienza en el componente React `[ComponenteReactEdicion]`. Este componente maneja el envío de datos modificados mediante un manejador de eventos.

```javascript
// [ComponenteReactEdicion].jsx
import React, { useState } from 'react';

export const ComponenteEdicion = ({ recursoInicial }) => {
  const [formData, setFormData] = useState(recursoInicial);

  const handleChange = (e) => {
    setFormData({ ...formData, [e.target.name]: e.target.value });
  };

  const handleSubmit = async (e) => {
    e.preventDefault();
    try {
      const response = await fetchActualizarRecurso(formData.id, formData);
      console.log('Recurso actualizado:', response);
    } catch (error) {
      console.error('Error al actualizar el recurso:', error);
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <input name="campoUno" value={formData.campoUno} onChange={handleChange} />
      <input name="campoDos" value={formData.campoDos} onChange={handleChange} />
      <button type="submit">Actualizar</button>
    </form>
  );
};
```

### 2. Red (HTTP Call)
La llamada de red se realiza mediante una petición HTTP PUT.

- **Método HTTP:** `PUT`
- **Ruta/URL:** `[EndpointUrl]/{id}` (Ejemplo: `/api/v1/recursos/123`)
- **Cabeceras (Headers):** `Content-Type: application/json`
- **Cuerpo (Request Body):** JSON con los campos actualizados.

```json
{
  "campoUno": "Nuevo Valor 1",
  "campoDos": "Nuevo Valor 2"
}
```

### 3. Controller (Spring Boot)
El controlador `[ControllerClass]` recibe la solicitud PUT, valida el cuerpo y el ID de ruta, y delega al servicio.

```java
// [ControllerClass].java
@RestController
@RequestMapping("/api/v1/recursos")
public class ControllerClass {

    private final ServiceClass serviceClass;

    public ControllerClass(ServiceClass serviceClass) {
        this.serviceClass = serviceClass;
    }

    @PutMapping("/{id}")
    public ResponseEntity<JPAEntity> actualizarRecurso(@PathVariable Long id, @Valid @RequestBody DTORequest dto) {
        JPAEntity recursoActualizado = serviceClass.actualizarRecurso(id, dto);
        return ResponseEntity.ok(recursoActualizado);
    }
}
```

### 4. Service
El servicio `[ServiceClass]` recupera la entidad existente, actualiza sus campos y guarda los cambios persistiendo la transacción.

```java
// [ServiceClass].java
@Service
public class ServiceClass {

    private final RepositoryInterface repositoryInterface;

    public ServiceClass(RepositoryInterface repositoryInterface) {
        this.repositoryInterface = repositoryInterface;
    }

    @Transactional
    public JPAEntity actualizarRecurso(Long id, DTORequest dto) {
        JPAEntity entidadExistente = repositoryInterface.findById(id)
                .orElseThrow(() -> new ResourceNotFoundException("No se encontró recurso con id: " + id));
        
        // Actualizar valores
        entidadExistente.setCampoUno(dto.getCampoUno());
        entidadExistente.setCampoDos(dto.getCampoDos());
        
        // Se guarda automáticamente al final de la transacción o invocando explícitamente save()
        return repositoryInterface.save(entidadExistente);
    }
}
```

### 5. Repository & Entity
La interfaz `[RepositoryInterface]` y la entidad `[JPAEntity]` se mapean a la tabla de PostgreSQL correspondientemente.

```java
// [RepositoryInterface].java
@Repository
public interface RepositoryInterface extends JpaRepository<JPAEntity, Long> {
}

// [JPAEntity].java
@Entity
@Table(name = "tabla_recursos")
public class JPAEntity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "campo_uno")
    private String campoUno;

    @Column(name = "campo_dos")
    private String campoDos;

    // Getters y Setters
}
```

### 6. Return Path
El flujo de retorno es el siguiente:
1. **Base de Datos:** Ejecuta la sentencia SQL (`UPDATE tabla_recursos SET campo_uno = ?, campo_dos = ? WHERE id = ?`).
2. **Repository:** Devuelve la entidad actualizada.
3. **Service:** Finaliza la transacción y retorna el objeto a `[ControllerClass]`.
4. **Controller:** Retorna la respuesta HTTP con código `200 OK` y el cuerpo JSON actualizado.
5. **Red:** Envía la respuesta JSON al cliente.
6. **Frontend (React):** Procesa los datos actualizados y refresca el estado en pantalla con los nuevos valores.
