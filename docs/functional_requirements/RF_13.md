# RF 13: Gestión de estados académicos

- **Explicación del Requerimiento:**
  Permite a los administradores y docentes del sistema actualizar manualmente el estado de matrícula de un estudiante en una sección académica. Esta funcionalidad es crítica para registrar el abandono del curso, marcar una observación manual o regularizar un estado. Los estados disponibles para cada matrícula son:
  - **`ENROLLED` (Activo):** El estudiante cursa regularmente la asignatura.
  - **`AT_RISK` (Observado / En Riesgo):** El estudiante presenta alertas de inasistencias o bajo rendimiento.
  - **`WITHDRAWN` (Retirado):** El estudiante ha sido dado de baja o anulado administrativamente de la sección de clase.
  
  Cualquier alteración o cambio entre los estados académicos genera de manera síncrona el envío de una alerta/notificación dirigida a la bandeja personal del estudiante correspondiente.

  2. **Capa de Red:**
     - **HTTP Method:** `PUT`
     - **Base URL:** `http://localhost:8081/api`
     - **Path:** `/enrollments/{id}/status`
     - **Request parameters:** 
       - `id` en la ruta como `@PathVariable` de tipo `Long`.
       - `status` en la URL de consulta (`@RequestParam`) con valores `ENROLLED`, `WITHDRAWN` o `AT_RISK`.
     - **Request body format:** Vacío.
     - **Cabeceras obligatorias:** `Authorization: Bearer <token>` con rol `ADMIN` o `TEACHER`.

       @PutMapping("/{id}/status")
       @PreAuthorize("hasAnyRole('ADMIN','TEACHER')")
       @Operation(summary = "Actualizar estado de matrícula (ADMIN / TEACHER)", description = "Estados permitidos: ENROLLED, WITHDRAWN, AT_RISK")
       public ResponseEntity<EnrollmentDTO> updateStatus(
               @PathVariable Long id,

  4. **Capa de Negocio (Backend Service):**

       ```java
       public EnrollmentDTO updateStatus(Long enrollmentId, EnrollmentStatus status) {
           Enrollment enrollment = enrollmentRepository.findById(enrollmentId)
                   .orElseThrow(() -> new ResourceNotFoundException("Matrícula no encontrada con id: " + enrollmentId));
           
           EnrollmentStatus oldStatus = enrollment.getStatus();
           enrollment.setStatus(status);
           Enrollment saved = enrollmentRepository.save(enrollment);

           if (oldStatus != status) {
               if (status == EnrollmentStatus.AT_RISK) {
                   notificationService.sendNotification(saved.getStudent(), 
                           "Actualización de Estado: Tu matrícula en la sección " + saved.getSection().getCode() + " ha sido cambiada a 'En Riesgo/Observado'.");
               } else if (status == EnrollmentStatus.ENROLLED) {
                   notificationService.sendNotification(saved.getStudent(), 
                           "Actualización de Estado: Tu matrícula en la sección " + saved.getSection().getCode() + " ahora figura como 'Activo'.");
               } else if (status == EnrollmentStatus.WITHDRAWN) {
                   notificationService.sendNotification(saved.getStudent(), 
                           "Actualización de Estado: Has sido registrado como 'Retirado' en la sección " + saved.getSection().getCode() + ".");
               }
           }

           return enrollmentMapper.toDto(saved);
       }
       ```

  5. **Capa de Persistencia (Repository & Entity):**
     - **Repository:** [EnrollmentRepository](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/persistance/repository/EnrollmentRepository.java)
     - **Spring Data JPA Methods:** `findById`, `save`.
     - **JPA Entity:** [Enrollment](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/persistance/entity/Enrollment.java) (tabla `enrollments`), mapped with [EnrollmentStatus](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/persistance/entity/EnrollmentStatus.java) enum.
     - **Funcionamiento de Base de Datos:**
       La persistencia ejecuta una consulta UPDATE: `UPDATE enrollments SET status = ? WHERE id = ?`. El motor de persistencia JDBC actualiza el campo `status` del registro en base de datos.

  6. **Camino de Retorno:**
     - El servicio retorna la matrícula actualizada en un `EnrollmentDTO`.
     - El controlador `EnrollmentController` serializa la respuesta JSON con HTTP 200 OK.
     - En el frontend ([Enrollments.jsx](file:///d:/Workspace/utp/EduTrack/Frontend/src/components/Enrollments.jsx)), la promesa de la petición se resuelve, se ejecuta `setSuccess(...)` para notificar del éxito en la esquina superior de la interfaz, y se invoca `fetchEnrollments(selectedSectionId)` para actualizar y sincronizar de forma inmediata la lista de alumnos en el DOM.
