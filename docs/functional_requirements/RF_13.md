# Requerimiento Funcional: RF 13 - Gestión de estados académicos (Plantilla Educativa)

## Explicación del Requerimiento (Genérico)
Este requerimiento define el flujo de comunicación estándar para la actualización del estado académico de un estudiante (por ejemplo, cambiar su estado de "Regular" a "En Riesgo" o "Suspendido"). El usuario administrador realiza la modificación en la interfaz gráfica, y los cambios se guardan de forma persistente en PostgreSQL a través del backend, validando la existencia previa del registro y aplicando reglas de negocio.

---

## Estructura del Flujo de Comunicación (Plantilla educativa)

### 1. Frontend (React)
El flujo comienza en el componente React `[ComponenteReact]`, el cual presenta un formulario o menú desplegable para cambiar el estado de un registro cargado previamente.

```javascript
// [ComponenteReact].jsx
import React, { useState } from 'react';

export const ComponenteEstadoAcademico = ({ estadoInicial }) => {
  const [estadoAcademico, setEstadoAcademico] = useState(estadoInicial);

  const handleActualizarEstado = async (e) => {
    e.preventDefault();
    try {
      const response = await fetchActualizarEstado(estadoAcademico.id, estadoAcademico);
      console.log('Estado académico actualizado:', response);
    } catch (error) {
      console.error('Error al actualizar estado académico:', error);
    }
  };

  const handleSelectChange = (e) => {
    setEstadoAcademico({ ...estadoAcademico, estado: e.target.value });
  };

  return (
    <form onSubmit={handleActualizarEstado}>
      <select value={estadoAcademico.estado} onChange={handleSelectChange}>
        <option value="REGULAR">Regular</option>
        <option value="EN_RIESGO">En Riesgo</option>
        <option value="CONDICIONAL">Condicional</option>
      </select>
      <button type="submit">Actualizar Estado</button>
    </form>
  );
};
```

### 2. Red (HTTP Call)
La llamada de red se ejecuta como una petición HTTP PUT o PATCH enviando el identificador y el nuevo estado en formato JSON.

- **Método HTTP:** `PUT`
- **Ruta/URL:** `[EndpointUrl]/{id}` (Ejemplo: `/api/v1/estados-academicos/123`)
- **Cabeceras (Headers):** `Content-Type: application/json`
- **Cuerpo (Request Body):**
```json
{
  "estado": "EN_RIESGO"
}
```

### 3. Controller (Spring Boot)
El controlador `[ControllerClass]` expone el endpoint PUT, mapea la solicitud a un DTO de validación y la delega al servicio.

```java
// [ControllerClass].java
@RestController
@RequestMapping("/api/v1/estados-academicos")
public class ControllerClass {

    private final ServiceClass serviceClass;

    public ControllerClass(ServiceClass serviceClass) {
        this.serviceClass = serviceClass;
    }

    @PutMapping("/{id}")
    public ResponseEntity<JPAEntity> actualizarEstado(@PathVariable Long id, @Valid @RequestBody DTORequest dto) {
        JPAEntity actualizado = serviceClass.actualizarEstado(id, dto);
        return ResponseEntity.ok(actualizado);
    }
}
```

### 4. Service
El servicio `[ServiceClass]` recupera la entidad correspondiente a través de su ID, aplica las validaciones del cambio de estado y actualiza el campo persistiendo la transacción.

```java
// [ServiceClass].java
@Service
public class ServiceClass {

    private final RepositoryInterface repositoryInterface;

    public ServiceClass(RepositoryInterface repositoryInterface) {
        this.repositoryInterface = repositoryInterface;
    }

    @Transactional
    public JPAEntity actualizarEstado(Long id, DTORequest dto) {
        JPAEntity entidadExistente = repositoryInterface.findById(id)
                .orElseThrow(() -> new ResourceNotFoundException("No se encontró estado académico para el ID: " + id));

        // Aplicar cambio
        entidadExistente.setEstado(dto.getEstado());

        return repositoryInterface.save(entidadExistente);
    }
}
```

### 5. Repository & Entity
La interfaz `[RepositoryInterface]` y la clase de entidad `[JPAEntity]` mapean el estado académico del estudiante a PostgreSQL mediante Spring Data JPA.

```java
// [RepositoryInterface].java
@Repository
public interface RepositoryInterface extends JpaRepository<JPAEntity, Long> {
}

// [JPAEntity].java
@Entity
@Table(name = "tabla_estados_academicos")
public class JPAEntity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "estado", nullable = false)
    private String estado;

    // Getters y Setters
}
```

### 6. Return Path
El flujo de retorno es el siguiente:
1. **Base de Datos:** Ejecuta la consulta SQL `UPDATE` y actualiza la fila correspondiente en la base de datos PostgreSQL.
2. **Repository:** Devuelve el objeto mapeado actualizado a `[ServiceClass]`.
3. **Service:** Finaliza la transacción y retorna el objeto a `[ControllerClass]`.
4. **Controller:** Envuelve el resultado en un `ResponseEntity` con código `200 OK` y envía la respuesta.
5. **Red:** Transmite el JSON de la entidad modificada al navegador cliente.
6. **Frontend (React):** `fetchActualizarEstado` resuelve la promesa de red, actualiza el estado de la UI y notifica al usuario del éxito del cambio.
