# RF 05: Asignación de docentes

- **Explicación del Requerimiento**:
  Este requerimiento permite asignar un docente específico a una sección académica para que asuma la tutoría y dictado de la clase. El administrador (rol `ADMIN`) es el único usuario con privilegios para realizar esta asignación. El usuario asignado como docente debe estar activo y poseer obligatoriamente el rol `TEACHER`. Una vez asignado, el docente podrá publicar materiales, registrar asistencia, y calificar tareas de la sección asignada.

- **Flujo de Código Paso a Paso**:

  ### 1. Interfaz de Usuario (Frontend)
  El componente React responsable de la interfaz de asignación de docentes es [Sections.jsx](file:///d:/Workspace/utp/EduTrack/Frontend/src/components/Sections.jsx). Cuando el administrador abre el modal de asignación, selecciona un docente del listado desplegable y hace clic en "Confirmar Asignación", se ejecuta `handleTeacherAssignSubmit`:

  ```javascript
  const handleTeacherAssignSubmit = async (e) => {
    e.preventDefault();
    if (!isAdmin) return;
    if (!selectedTeacherId) {
      setError('Por favor seleccione un docente');
      return;
    }

    setError('');
    setSuccess('');

    try {
      await api.sections.assignTeacher(assignSectionId, parseInt(selectedTeacherId));
      setSuccess('Docente asignado con éxito a la sección');
      setShowTeacherModal(false);
      fetchData();
    } catch (err) {
      setError(err.message || 'Error al asignar docente');
    }
  };
  ```

  La llamada de red utiliza el servicio encapsulado en [api.js](file:///d:/Workspace/utp/EduTrack/Frontend/src/services/api.js):

  ```javascript
  export const api = {
    sections: {
      assignTeacher: (sectionId, teacherId) => request(`/sections/${sectionId}/teacher`, {
        method: 'PUT',
        body: JSON.stringify({ teacherId }),
      }),
      // ...
    }
  };
  ```

  ### 2. Capa de Red
  - **Método HTTP**: `PUT`
  - **URL Base**: `http://localhost:8081/api`
  - **Ruta (Path)**: `/sections/{id}/teacher` (donde `{id}` es el identificador numérico de la sección en base de datos, por ejemplo `/sections/1/teacher`).
  - **Parámetros**: Ninguno en la URL query.
  - **Cuerpo de la Petición (Request Body)**: Formato JSON.
    ```json
    {
      "teacherId": 3
    }
    ```

  ### 3. Controlador (Backend Controller)
  La petición es recibida por [SectionController.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/web/controller/SectionController.java), controlado por permisos de administrador `@PreAuthorize("hasRole('ADMIN')")`:

  ```java
  @RestController
  @RequestMapping("/api/sections")
  @RequiredArgsConstructor
  @SecurityRequirement(name = OpenApiConfig.BEARER_AUTH)
  @Tag(name = "Secciones", description = "RF04 y RF05 - Lectura: todos. Gestión: solo ADMIN")
  public class SectionController {

      private final SectionService sectionService;

      @PutMapping("/{id}/teacher")
      @PreAuthorize("hasRole('ADMIN')")
      @Operation(summary = "Asignar docente (ADMIN) (RF05)", description = "Campo: teacherId (usuario con rol TEACHER)")
      public ResponseEntity<SectionDTO> assignTeacher(
              @PathVariable Long id,
              @Valid @RequestBody AssignTeacherDTO request
      ) {
          return ResponseEntity.ok(sectionService.assignTeacher(id, request));
      }
  }
  ```

  ### 4. Capa de Negocio (Backend Service)
  La lógica de asignación se gestiona en [SectionService.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/domain/service/SectionService.java). El método valida la existencia del docente y la sección, confirmando que el usuario tenga rol de docente y esté en estado activo:

  ```java
  public SectionDTO assignTeacher(Long sectionId, AssignTeacherDTO request) {
      Section section = getSectionEntity(sectionId);
      User teacher = userService.getUserEntity(request.getTeacherId());

      if (teacher.getRole() != Role.TEACHER) {
          throw new BusinessException("El usuario asignado debe tener rol TEACHER");
      }
      if (!teacher.isActive()) {
          throw new BusinessException("El docente debe estar activo");
      }

      section.setTeacher(teacher);
      return sectionMapper.toDto(sectionRepository.save(section));
  }
  ```

  ### 5. Capa de Persistencia (Repository & Entity)
  - **Repositorios**:
    - [UserRepository.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/persistance/repository/UserRepository.java): Recupera la entidad `User` usando `findById(teacherId)`.
    - [SectionRepository.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/persistance/repository/SectionRepository.java): Recupera la entidad `Section` usando `findById(sectionId)` y la persiste llamando a `save(section)`.
  - **Entidades**:
    - [Section.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/persistance/entity/Section.java), que tiene una relación `@ManyToOne` con la entidad [User.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/persistance/entity/User.java) a través de la columna `teacher_id`:
      ```java
      @ManyToOne(fetch = FetchType.LAZY)
      @JoinColumn(name = "teacher_id")
      private User teacher;
      ```
    El motor de base de datos ejecuta una sentencia de actualización:
    `UPDATE sections SET teacher_id = ? WHERE id = ?`

  ### 6. Camino de Retorno
  1. El repositorio persiste el cambio de clave foránea en la base de datos y retorna el objeto `Section`.
  2. El servicio mapea la entidad actualizada a un `SectionDTO` (el cual incluye campos como `teacherUsername` y `teacherId`) y lo retorna al controlador.
  3. El controlador responde al cliente con un estado `200 OK` y el cuerpo serializado del `SectionDTO`.
  4. En frontend, `api.js` procesa y deserializa la respuesta resolviendo exitosamente la promesa.
  5. En `Sections.jsx`, el método `handleTeacherAssignSubmit` despliega la notificación de éxito (`setSuccess('Docente asignado con éxito a la sección')`), cierra el modal (`setShowTeacherModal(false)`) y gatilla `fetchData()`.
  6. `fetchData()` realiza la petición GET `/api/sections` mediante `api.sections.getAll()`. Al obtener las secciones, actualiza el estado local de React a través de `setSections(sectionsData)`, repintando el listado en la pantalla y mostrando el nombre del nuevo docente en la columna "Docente Asignado" de la fila de la sección.
