# Requerimiento Funcional: RF 09 - Registro de asistencia (Plantilla Educativa)

## Explicación del Requerimiento (Genérico)
Este requerimiento define el flujo genérico para registrar y actualizar el estado de asistencia de los estudiantes matriculados en una sesión, clase o semana académica. El docente selecciona un grupo/curso, una fecha o número de sesión, y asigna a cada estudiante un estado de asistencia predefinido (ejemplo: Presente, Ausente, Tardanza, Justificado). Esta información es guardada en PostgreSQL para llevar el control de la tasa de asistencia histórica.

## Estructura del Flujo de Comunicación (Plantilla educativa)

### 1. Frontend (React)
El componente React `[ComponenteReact]` despliega una grilla con los nombres de los estudiantes y opciones de selección (select/botones) para su estado. Al cambiar el estado de un alumno, se gatilla la petición HTTP.

```javascript
// [ComponenteReact].jsx
import React, { useState } from 'react';

const [ComponenteReact] = ({ estudianteId, sesionId }) => {
  const [estado, setEstado] = useState('PRESENTE'); // PRESENTE, AUSENTE, TARDE

  const handleChangeAsistencia = async (nuevoEstado) => {
    setEstado(nuevoEstado);
    try {
      const response = await fetch('[EndpointUrl]', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': 'Bearer [TokenDeAutenticacion]'
        },
        body: JSON.stringify({ estudianteId, sesionId, estado: nuevoEstado })
      });

      if (!response.ok) {
        throw new Error('Error al actualizar la asistencia');
      }

      const data = await response.json();
      console.log('Asistencia guardada:', data);
    } catch (err) {
      console.error('Error al registrar asistencia:', err);
    }
  };

  return (
    <div>
      <span>Estudiante ID: {estudianteId}</span>
      <select value={estado} onChange={(e) => handleChangeAsistencia(e.target.value)}>
        <option value="PRESENTE">Presente</option>
        <option value="AUSENTE">Ausente</option>
        <option value="TARDE">Tarde</option>
      </select>
    </div>
  );
};

export default [ComponenteReact];
```

### 2. Red (HTTP Call)
Los datos de asistencia viajan como payload de la petición HTTP:
- **Método HTTP**: `POST` (o `PUT` para actualización de asistencia)
- **Ruta de la API**: `[EndpointUrl]` (Ejemplo: `/api/v1/asistencias`)
- **Encabezados (Headers)**:
  - `Content-Type: application/json`
  - `Authorization: Bearer [TokenDeAutenticacion]`
- **Cuerpo del Mensaje (Request Body)**:
  ```json
  {
    "estudianteId": 101,
    "sesionId": 502,
    "estado": "AUSENTE"
  }
  ```

### 3. Controller (Spring Boot)
La clase controladora `[ControllerClass]` maneja la entrada de datos por HTTP y delega la ejecución de lógica al servicio de asistencia.

```java
// [ControllerClass].java
package [PackageController];

import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("[EndpointUrl]")
public class [ControllerClass] {

    private final [ServiceClass] service;

    public [ControllerClass]([ServiceClass] service) {
        this.service = service;
    }

    @PostMapping
    public ResponseEntity<[DTOClass]> registrarAsistencia(@RequestBody [CreateDTOClass] requestDto) {
        [DTOClass] result = service.guardarAsistencia(requestDto);
        return ResponseEntity.status(HttpStatus.OK).body(result);
    }
}
```

### 4. Service
La clase de servicio `[ServiceClass]` contiene la lógica de negocio. Verifica las condiciones necesarias (por ejemplo, si el periodo académico o la sesión están activos para edición) y actualiza o crea el registro en la base de datos PostgreSQL.

```java
// [ServiceClass].java
package [PackageService];

import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
public class [ServiceClass] {

    private final [RepositoryInterface] repository;

    public [ServiceClass]([RepositoryInterface] repository) {
        this.repository = repository;
    }

    @Transactional
    public [DTOClass] guardarAsistencia([CreateDTOClass] dto) {
        // Validar negocio (ej: si el docente tiene permiso sobre esa clase o sesión)
        
        // Buscar si ya existe el registro de asistencia para este alumno en esta sesión
        [JPAEntity] entity = repository
            .findByEstudianteIdAndSesionId(dto.getEstudianteId(), dto.getSesionId())
            .orElse(new [JPAEntity]());

        entity.setEstudianteId(dto.getEstudianteId());
        entity.setSesionId(dto.getSesionId());
        entity.setEstado(dto.getEstado()); // Enum o String (PRESENTE, AUSENTE, TARDE)
        entity.setFechaRegistro(java.time.LocalDateTime.now());

        [JPAEntity] saved = repository.save(entity);

        // Lógica adicional opcional: Notificar al estudiante si está AUSENTE
        // if ("AUSENTE".equals(dto.getEstado())) { [NotificacionService].enviarAlerta(dto.getEstudianteId()); }

        return mapToDto(saved);
    }

    private [DTOClass] mapToDto([JPAEntity] entity) {
        [DTOClass] dto = new [DTOClass]();
        dto.setId(entity.getId());
        dto.setEstudianteId(entity.getEstudianteId());
        dto.setSesionId(entity.getSesionId());
        dto.setEstado(entity.getEstado());
        dto.setFechaRegistro(entity.getFechaRegistro());
        return dto;
    }
}
```

### 5. Repository & Entity
- **Repository**: La interfaz `[RepositoryInterface]` extiende de `JpaRepository` y define búsquedas basadas en la sesión y el estudiante.
- **Entity**: La clase `[JPAEntity]` mapea los atributos del registro de asistencia con la tabla correspondiente en PostgreSQL.

```java
// [RepositoryInterface].java

@Repository

    Optional<[JPAEntity]> findByEstudianteIdAndSesionId(Long estudianteId, Long sesionId);

```

### 6. Return Path
1. **PostgreSQL**: Registra/Actualiza la tupla en la tabla y devuelve la confirmación.
2. **[RepositoryInterface]**: Retorna el objeto `[JPAEntity]` guardado.
3. **[ServiceClass]**: Adapta el objeto `[JPAEntity]` a `[DTOClass]` y lo envía de regreso a la capa web.
4. **[ControllerClass]**: Responde HTTP `200 OK` retornando el DTO convertido en formato JSON.
5. **Red**: Transporta los datos de vuelta a la aplicación cliente React.
6. **[ComponenteReact]**: Recibe el JSON y actualiza el estado del estudiante en la tabla local para reflejar el cambio de estado (cambio de color, actualización del contador total de asistencias, etc.).
