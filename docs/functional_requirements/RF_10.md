# Requerimiento Funcional: RF 10 - Justificación de inasistencias (Plantilla Educativa)

## Explicación del Requerimiento (Genérico)
Este requerimiento define el flujo de solicitud y aprobación para justificar inasistencias (estados "AUSENTE").
El flujo educativo abarca dos roles:
1. **Estudiante (Solicitante)**: Envía una justificación adjuntando una descripción/motivo y opcionalmente la ruta de un documento de soporte (médico, laboral, etc.), registrándose con estado "PENDIENTE".
2. **Docente o Administrador (Aprobador)**: Visualiza las solicitudes pendientes de su clase y decide "APROBAR" o "RECHAZAR" la solicitud. Si se aprueba, el sistema actualiza automáticamente el registro de asistencia del estudiante a "JUSTIFICADO".

## Estructura del Flujo de Comunicación (Plantilla educativa)

### 1. Frontend (React)
El componente React `[ComponenteReact]` gestiona dos vistas: la creación de la justificación por el estudiante y la tabla de revisión de justificaciones por el docente.

```javascript
// [ComponenteReact].jsx
import React, { useState } from 'react';

const [ComponenteReact] = ({ inasistenciaId }) => {
  const [motivo, setMotivo] = useState('');
  const [documentoSustento, setDocumentoSustento] = useState('');

  const handleCreateJustificacion = async (e) => {
    e.preventDefault();
    try {
      const response = await fetch('[EndpointUrl]', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': 'Bearer [TokenDeAutenticacion]'
        },
        body: JSON.stringify({ inasistenciaId, motivo, documentoSustento })
      });

      if (!response.ok) {
        throw new Error('Error al enviar la solicitud de justificación');
      }

      const data = await response.json();
      alert('Solicitud enviada. Estado: PENDIENTE');
    } catch (err) {
      console.error('Error al solicitar justificación:', err);
    }
  };

  return (
    <form onSubmit={handleCreateJustificacion}>
      <textarea value={motivo} onChange={(e) => setMotivo(e.target.value)} placeholder="Motivo de la falta..." />
      <input type="text" value={documentoSustento} onChange={(e) => setDocumentoSustento(e.target.value)} placeholder="URL del documento..." />
      <button type="submit">Enviar Solicitud</button>
    </form>
  );
};

export default [ComponenteReact];
```

### 2. Red (HTTP Call)
Se transmite en formato JSON:
- **Método HTTP**: `POST` (o `PATCH`/`PUT` para la acción de aprobar/rechazar)
- **Ruta de la API**: `[EndpointUrl]` (Ejemplo: `/api/v1/justificaciones`)
- **Encabezados (Headers)**:
  - `Content-Type: application/json`
  - `Authorization: Bearer [TokenDeAutenticacion]`
- **Cuerpo del Mensaje (Request Body) - Crear Solicitud**:
  ```json
  {
    "inasistenciaId": 890,
    "motivo": "Cita médica programada",
    "documentoSustento": "http://storage.com/certificados/medico101.pdf"
  }
  ```
- **Cuerpo del Mensaje (Request Body) - Resolver Solicitud (PATCH `[EndpointUrl]/{id}/resolver`)**:
  ```json
  {
    "aprobado": true,
    "comentarioResolucion": "Certificado médico válido."
  }
  ```

### 3. Controller (Spring Boot)
La clase `[ControllerClass]` expone los métodos para crear la solicitud (estudiante) y para resolverla (docente/administrador).

```java
// [ControllerClass].java
package [PackageController];

import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("[EndpointUrl]")
public class [ControllerClass] {

    private final [ServiceClass] service;

    public [ControllerClass]([ServiceClass] service) {
        this.service = service;
    }

    @PostMapping
    public ResponseEntity<[DTOClass]> solicitarJustificacion(@RequestBody [CreateDTOClass] requestDto) {
        [DTOClass] created = service.crearSolicitud(requestDto);
        return ResponseEntity.status(HttpStatus.CREATED).body(created);
    }

    @PatchMapping("/{id}/resolver")
    public ResponseEntity<[DTOClass]> resolverJustificacion(
            @PathVariable("id") Long id,
            @RequestBody [ResolveDTOClass] resolveDto) {
        [DTOClass] resolved = service.resolverSolicitud(id, resolveDto);
        return ResponseEntity.ok(resolved);
    }
}
```

### 4. Service
La clase de servicio `[ServiceClass]` maneja la lógica compleja de negocio y coordina múltiples repositorios. Al aprobarse, debe alterar el estado de la asistencia correspondiente.

```java
// [ServiceClass].java
package [PackageService];

import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
public class [ServiceClass] {

    private final [RepositoryInterface] repository;
    private final [AsistenciaRepositoryInterface] asistenciaRepository;

    public [ServiceClass]([RepositoryInterface] repository, [AsistenciaRepositoryInterface] asistenciaRepository) {
        this.repository = repository;
        this.asistenciaRepository = asistenciaRepository;
    }

    @Transactional
    public [DTOClass] crearSolicitud([CreateDTOClass] dto) {
        [JPAEntity] entity = new [JPAEntity]();
        entity.setInasistenciaId(dto.getInasistenciaId());
        entity.setMotivo(dto.getMotivo());
        entity.setDocumentoSustento(dto.getDocumentoSustento());
        entity.setEstado("PENDIENTE");
        entity.setFechaCreacion(java.time.LocalDateTime.now());

        [JPAEntity] saved = repository.save(entity);
        return mapToDto(saved);
    }

    @Transactional
    public [DTOClass] resolverSolicitud(Long id, [ResolveDTOClass] dto) {
        [JPAEntity] entity = repository.findById(id)
                .orElseThrow(() -> new IllegalArgumentException("Solicitud no encontrada"));

        if (!"PENDIENTE".equals(entity.getEstado())) {
            throw new IllegalStateException("Esta solicitud ya ha sido resuelta");
        }

        if (dto.isAprobado()) {
            entity.setEstado("APROBADO");
            
            // Lógica cruzada: Cambiar la asistencia asociada a "JUSTIFICADO"
            [AsistenciaJPAEntity] asistencia = asistenciaRepository.findById(entity.getInasistenciaId())
                    .orElseThrow(() -> new IllegalArgumentException("Asistencia relacionada no existe"));
            asistencia.setEstado("JUSTIFICADO");
            asistenciaRepository.save(asistencia);
        } else {
            entity.setEstado("RECHAZADO");
        }

        entity.setFechaResolucion(java.time.LocalDateTime.now());
        entity.setComentarioResolucion(dto.getComentarioResolucion());

        [JPAEntity] saved = repository.save(entity);
        return mapToDto(saved);
    }

    private [DTOClass] mapToDto([JPAEntity] entity) {
        [DTOClass] dto = new [DTOClass]();
        dto.setId(entity.getId());
        dto.setInasistenciaId(entity.getInasistenciaId());
        dto.setMotivo(entity.getMotivo());
        dto.setDocumentoSustento(entity.getDocumentoSustento());
        dto.setEstado(entity.getEstado());
        dto.setFechaCreacion(entity.getFechaCreacion());
        dto.setFechaResolucion(entity.getFechaResolucion());
        dto.setComentarioResolucion(entity.getComentarioResolucion());
        return dto;
    }
}
```

### 5. Repository & Entity
- **Repository**: Las interfaces `[RepositoryInterface]` (para solicitudes) y `[AsistenciaRepositoryInterface]` (para el estado de asistencia) heredan de `JpaRepository` para el acceso a PostgreSQL.
- **Entity**: La clase `[JPAEntity]` mapea los campos de auditoría y justificación en la base de datos.

```java
// [RepositoryInterface].java
package [PackageRepository];

import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.stereotype.Repository;

@Repository
public interface [RepositoryInterface] extends JpaRepository<[JPAEntity], Long> {
}
```

@Entity

### 6. Return Path
1. **PostgreSQL**: Modifica el registro de la justificación y actualiza opcionalmente el estado de la asistencia a "JUSTIFICADO" (usando llaves foráneas o transacciones lógicas), confirmando los cambios.
2. **[RepositoryInterface]** & **[AsistenciaRepositoryInterface]**: Retornan las entidades JPA debidamente actualizadas.
3. **[ServiceClass]**: Traduce el resultado final de `[JPAEntity]` a un objeto DTO limpio de tipo `[DTOClass]` de respuesta.
4. **[ControllerClass]**: Empaqueta el DTO en un `ResponseEntity.ok()` (estado HTTP 200) y el servidor HTTP serializa el payload a JSON.
5. **Red**: Transmite la respuesta JSON desde el backend al navegador del cliente.
6. **[ComponenteReact]**: Recibe la confirmación en el frontend, actualiza dinámicamente el estado local de la justificación en pantalla (de "PENDIENTE" a "APROBADO"/"RECHAZADO"), y actualiza el contador o estado visual de la asistencia asociada en la UI del usuario.
