# RF 06: Gestión de evaluaciones

- **Explicación del Requerimiento (Genérico)**:
  Este requerimiento define la creación, lectura y actualización de evaluaciones asociadas a cursos, asignaturas o secciones en el sistema. Permite a los usuarios con roles docentes o administrativos estructurar las actividades académicas del periodo escolar, especificando atributos esenciales tales como el título de la evaluación, descripción, fecha máxima de entrega, límite de intentos y la ponderación (peso porcentual) de la misma.

- **Estructura del Flujo de Comunicación (Plantilla educativa)**:

  ### 1. Frontend (React)
  El componente de la interfaz de usuario `[ComponenteReact]` maneja la interacción del usuario y la captura de datos del formulario. Al enviar el formulario de registro, llama a la API.

  ```javascript
  // [ComponenteReact].jsx
  import React, { useState } from 'react';

  const [ComponenteReact] = () => {
    const [evaluacionData, setEvaluacionData] = useState({
      titulo: '',
      descripcion: '',
      fechaLimite: '',
      maxIntentos: 1,
      ponderacion: 0
    });

    const handleSave = async (e) => {
      e.preventDefault();
      try {
        const response = await fetch('[EndpointUrl]', {
          method: 'POST',
          headers: {
            'Content-Type': 'application/json',
            'Authorization': 'Bearer [TokenDeAutenticacion]'
          },
          body: JSON.stringify(evaluacionData)
        });

        if (!response.ok) {
          throw new Error('Error al procesar la solicitud');
        }

        const data = await response.json();
        alert('Evaluación guardada con éxito');
        // Limpiar el estado o actualizar la interfaz local
      } catch (err) {
        console.error('Error en el frontend:', err);
      }
    };

    return (
      <form onSubmit={handleSave}>
        {/* Renderizado de inputs genéricos de título, descripción, etc. */}
      </form>
    );
  };

  export default [ComponenteReact];
  ```

  ### 2. Red (HTTP Call)
  La transferencia de datos se realiza a través de una petición HTTP asíncrona:
  - **Método HTTP**: `POST`
  - **Ruta de la API**: `[EndpointUrl]` (Ejemplo: `/api/v1/evaluaciones`)
  - **Encabezados (Headers)**:
    - `Content-Type: application/json`
    - `Authorization: Bearer [TokenDeAutenticacion]`
  - **Cuerpo del Mensaje (Request Body)**:
    ```json
    {
      "titulo": "[TituloDeEvaluacion]",
      "descripcion": "[DescripcionDeEvaluacion]",
      "fechaLimite": "2026-12-31T23:59:59",
      "maxIntentos": 2,
      "ponderacion": 15
    }
    ```

  ### 3. Controller (Spring Boot)
  La clase controladora `[ControllerClass]` recibe la petición web, valida el cuerpo JSON de entrada utilizando anotaciones estándar de validación y delega el flujo a la capa de negocio.

  ```java
  // [ControllerClass].java
  package [PackageController];

  import org.springframework.http.HttpStatus;
  import org.springframework.http.ResponseEntity;
  import org.springframework.web.bind.annotation.*;

  @RestController
  @RequestMapping("[EndpointUrl]")
  public class [ControllerClass] {

      private final [ServiceClass] service;

      public [ControllerClass]([ServiceClass] service) {
          this.service = service;
      }

      @PostMapping
      public ResponseEntity<[DTOClass]> create(@RequestBody [CreateDTOClass] requestDto) {
          [DTOClass] created = service.create(requestDto);
          return ResponseEntity.status(HttpStatus.CREATED).body(created);
      }
  }
  ```

  ### 4. Service
  La clase de servicio `[ServiceClass]` contiene las validaciones de negocio adicionales (por ejemplo, comprobar que las fechas límite sean posteriores a la fecha actual y que la ponderación acumulada no supere el 100%).

  ```java
  // [ServiceClass].java
  package [PackageService];

  import org.springframework.stereotype.Service;
  import org.springframework.transaction.annotation.Transactional;

  @Service
  public class [ServiceClass] {

      private final [RepositoryInterface] repository;

      public [ServiceClass]([RepositoryInterface] repository) {
          this.repository = repository;
      }

      @Transactional
      public [DTOClass] create([CreateDTOClass] dto) {
          // Lógica de negocio: Validaciones complejas
          if (dto.getPonderacion() <= 0 || dto.getPonderacion() > 100) {
              throw new IllegalArgumentException("La ponderación debe ser entre 1 y 100.");
          }

          // Mapear DTO a Entidad
          [JPAEntity] entity = new [JPAEntity]();
          entity.setTitulo(dto.getTitulo());
          entity.setDescripcion(dto.getDescripcion());
          entity.setFechaLimite(dto.getFechaLimite());
          entity.setMaxIntentos(dto.getMaxIntentos());
          entity.setPonderacion(dto.getPonderacion());

          // Persistir en base de datos
          [JPAEntity] saved = repository.save(entity);

          // Retornar DTO mapeado
          return mapToDto(saved);
      }

      private [DTOClass] mapToDto([JPAEntity] entity) {
          [DTOClass] dto = new [DTOClass]();
          dto.setId(entity.getId());
          dto.setTitulo(entity.getTitulo());
          dto.setDescripcion(entity.getDescripcion());
          dto.setFechaLimite(entity.getFechaLimite());
          dto.setMaxIntentos(entity.getMaxIntentos());
          dto.setPonderacion(entity.getPonderacion());
          return dto;
      }
  }
  ```

  ### 5. Repository & Entity
  - **Repository**: La interfaz `[RepositoryInterface]` extiende de `JpaRepository<[JPAEntity], Long>` y gestiona la comunicación con la base de datos PostgreSQL mediante Spring Data JPA.
  - **Entity**: La clase `[JPAEntity]` mapea los atributos lógicos a las columnas de la tabla en base de datos.

  ```java
  // [RepositoryInterface].java
  package [PackageRepository];

  import org.springframework.data.jpa.repository.JpaRepository;
  import org.springframework.stereotype.Repository;

  @Repository
  public interface [RepositoryInterface] extends JpaRepository<[JPAEntity], Long> {
  }
  ```

  ```java
  // [JPAEntity].java
  package [PackageEntity];

  import jakarta.persistence.*;
  import java.time.LocalDateTime;

  @Entity
  @Table(name = "[NombreTablaEvaluacion]")
  public class [JPAEntity] {

      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;

      @Column(nullable = false, length = 150)
      private String titulo;

      @Column(columnDefinition = "TEXT")
      private String descripcion;

      @Column(name = "fecha_limite", nullable = false)
      private LocalDateTime fechaLimite;

      @Column(name = "max_intentos")
      private Integer maxIntentos;

      @Column(nullable = false)
      private Integer ponderacion;

      // Getters, Setters y Constructores
  }
  ```

  ### 6. Return Path
  1. **PostgreSQL**: Registra los datos de la evaluación y retorna la tupla generada con su clave primaria única (`id`).
  2. **[RepositoryInterface]**: Retorna la instancia de `[JPAEntity]` con su ID asignado.
  3. **[ServiceClass]**: Transforma la entidad `[JPAEntity]` a un objeto DTO limpio `[DTOClass]` de tipo respuesta y lo devuelve a la capa del controlador.
  4. **[ControllerClass]**: Envuelve el DTO en un objeto `ResponseEntity` con cabeceras correctas y un código de estado `201 Created`, entregándolo a la capa del servidor HTTP para su serialización a formato JSON.
  5. **Red**: El JSON viaja de regreso al cliente mediante HTTP.
  6. **[ComponenteReact]**: Procesa la respuesta JSON exitosa, limpia el estado de las entradas del formulario y actualiza la vista (o actualiza una lista de evaluaciones de forma dinámica en la interfaz de usuario).
