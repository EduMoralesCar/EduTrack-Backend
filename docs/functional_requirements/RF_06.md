# RF 06: Gestión de evaluaciones

- **Explicación del Requerimiento**:
  Este requerimiento permite a los usuarios con rol de docente (`TEACHER`) o administrador (`ADMIN`) gestionar (crear y actualizar) las evaluaciones y tareas asignadas a una sección específica de un curso. Cada evaluación pertenece a una categoría (PA, PC1, PC2, PC3, EXFINAL), se asocia a una semana del ciclo (1 a 18), tiene una fecha de inicio y una fecha límite de entrega, y un límite máximo de intentos permitidos. Opcionalmente, se puede cargar un archivo de indicaciones adjunto para los estudiantes.

       ```java
       @Transactional
       public AssignmentDTO createAssignment(AssignmentDTO dto, MultipartFile file) {
           Section section = sectionRepository.findById(dto.getSectionId())
                   .orElseThrow(() -> new ResourceNotFoundException("Section not found"));

           String instructionsPath = null;
           if (file != null && !file.isEmpty()) {
               instructionsPath = fileStorageService.storeFile(file);
           }

           Assignment assignment = Assignment.builder()
                   .name(dto.getName())
                   .type(dto.getType())
                   .startDate(dto.getStartDate())
                   .endDate(dto.getEndDate())
                   .weekNumber(dto.getWeekNumber())
                   .category(dto.getCategory())
                   .description(dto.getDescription())
                   .instructionsFilePath(instructionsPath)
                   .maxAttempts(dto.getMaxAttempts() != null ? dto.getMaxAttempts() : 1)
                   .section(section)
                   .build();

           Assignment saved = assignmentRepository.save(assignment);
           dto.setId(saved.getId());
           dto.setDescription(saved.getDescription());
           dto.setInstructionsFilePath(normalizeFilePath(saved.getInstructionsFilePath()));
           dto.setMaxAttempts(saved.getMaxAttempts());
           return dto;
       }
       ```

  5. **Capa de Persistencia (Repository & Entity)**:

     - **Entidad**: [Assignment.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/persistance/entity/Assignment.java)

       - **Detalles de Registro/Actualización/Consulta**: Se realiza un `INSERT` en la tabla `assignments` mapeando campos como `name`, `type` (como string), `start_date`, `end_date`, `week_number`, `category`, `description`, `instructions_file_path`, `max_attempts` y `section_id` (FK a la tabla `sections`).

  6. **Camino de Retorno**:
     - El backend responde con un código de estado HTTP `201 Created` y el objeto `AssignmentDTO` serializado a JSON que contiene los datos creados con su nuevo identificador único.
     - En el frontend, el helper `handleResponse` en `api.js` deserializa la respuesta.
     - El componente `Assignments.jsx` recibe este resultado en `handleCreateAssignment`, muestra un cuadro de diálogo con `alert('Evaluación creada exitosamente!')`, limpia el formulario con `e.target.reset()` y vuelve a llamar a `api.assignments.getBySection(selectedSection)` para actualizar el arreglo del estado local `assignments` mediante `setAssignments(asgs)`. Esto desencadena una re-renderización automática de la tabla de evaluaciones en la interfaz de usuario.
