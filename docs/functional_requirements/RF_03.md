# RF 03: Gestión de cursos

- **Explicación del Requerimiento**:
  Este requerimiento permite la administración (registro, modificación, eliminación y listado) de las materias o asignaturas académicas que se dictan en la institución. La capacidad de registrar, actualizar y eliminar cursos está reservada para el rol `ADMIN`. Los docentes y estudiantes pueden consultar el catálogo de asignaturas asignadas o matriculadas correspondientemente. Cada curso consta de un código único (ej. INF-401), un nombre descriptivo, créditos académicos, y una descripción detallada.

- **Flujo de Código Paso a Paso**:

  ### 1. Interfaz de Usuario (Frontend)
  El componente React encargado de la interfaz es [Courses.jsx](file:///d:/Workspace/utp/EduTrack/Frontend/src/components/Courses.jsx). Cuando un administrador envía el formulario de creación, se dispara la función `handleSubmit`:

  ```javascript
  const handleSubmit = async (e) => {
    e.preventDefault();
    if (!isAdmin) return;
    if (!name || !code || !credits) {
      setError('Por favor complete los campos requeridos');
      return;
    }

    setError('');
    setSuccess('');

    try {
      const courseData = { name, code, description, credits: parseInt(credits) };
      
      if (isEditing) {
        await api.courses.update(selectedCourseId, courseData);
        setSuccess('Curso actualizado con éxito');
      } else {
        await api.courses.create(courseData);
        setSuccess('Curso creado con éxito');
      }
      setShowModal(false);
      fetchCourses();
    } catch (err) {
      setError(err.message || 'Error al guardar el curso');
    }
  };
  ```

  La llamada de red utiliza el servicio encapsulado en [api.js](file:///d:/Workspace/utp/EduTrack/Frontend/src/services/api.js):

  ```javascript
  export const api = {
    courses: {
      create: (data) => request('/courses', {
        method: 'POST',
        body: JSON.stringify(data),
      }),
      // ...
    }
  };
  ```

  ### 2. Capa de Red
  - **Método HTTP**: `POST`
  - **URL Base**: `http://localhost:8081/api`
  - **Ruta (Path)**: `/courses`
  - **Parámetros**: Ninguno.
  - **Cuerpo de la Petición (Request Body)**: Formato JSON.
    ```json
    {
      "name": "Introducción a la Programación",
      "code": "INF-401",
      "description": "Conceptos básicos de algoritmos y programación estructurada.",
      "credits": 3
    }
    ```

  ### 3. Controlador (Backend Controller)
  La solicitud es capturada en [CourseController.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/web/controller/CourseController.java), controlado con `@PreAuthorize("hasRole('ADMIN')")`:

  ```java
  @RestController
  @RequestMapping("/api/courses")
  @RequiredArgsConstructor
  @SecurityRequirement(name = OpenApiConfig.BEARER_AUTH)
  @Tag(name = "Cursos", description = "RF03 - Lectura: todos los roles. Escritura: solo ADMIN")
  public class CourseController {

      private final CourseService courseService;

      @PostMapping
      @PreAuthorize("hasRole('ADMIN')")
      @Operation(summary = "Registrar curso (ADMIN)", description = "Campos: name, code, description, credits")
      public ResponseEntity<CourseDTO> create(@Valid @RequestBody CourseRequestDTO request) {
          return ResponseEntity.status(HttpStatus.CREATED).body(courseService.create(request));
      }
  }
  ```

  ### 4. Capa de Negocio (Backend Service)
  La lógica reside en [CourseService.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/domain/service/CourseService.java). Valida que el código del curso no esté duplicado, realiza el mapeo de DTO a entidad mediante `CourseMapper` y persiste en base de datos:

  ```java
  public CourseDTO create(CourseRequestDTO request) {
      validateUniqueCode(request.getCode(), null);
      Course saved = courseRepository.save(courseMapper.toEntity(request));
      return courseMapper.toDto(saved);
  }

  private void validateUniqueCode(String code, Long excludeId) {
      String normalized = code.trim().toUpperCase();
      boolean exists = excludeId == null
              ? courseRepository.existsByCode(normalized)
              : courseRepository.existsByCodeAndIdNot(normalized, excludeId);

      if (exists) {
          throw new BusinessException("El código de curso ya existe: " + normalized);
      }
  }
  ```

  ### 5. Capa de Persistencia (Repository & Entity)
  - **Repositorio**: [CourseRepository.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/persistance/repository/CourseRepository.java), extiende de `JpaRepository<Course, Long>`. Expone el método `existsByCode` para validar códigos únicos y hereda el método `save` para registrar.
  - **Entidad**: [Course.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/persistance/entity/Course.java), mapeada a la tabla `courses`.
    ```java
    @Entity
    @Table(name = "courses")
    @Getter
    @Setter
    @NoArgsConstructor
    @AllArgsConstructor
    @Builder
    public class Course {
        @Id
        @GeneratedValue(strategy = GenerationType.IDENTITY)
        private Long id;

        @Column(nullable = false, length = 150)
        private String name;

        @Column(nullable = false, unique = true, length = 20)
        private String code;

        @Column(length = 500)
        private String description;

        @Column(nullable = false)
        private Integer credits;
    }
    ```
    La persistencia realiza un query de inserción sql: `INSERT INTO courses (name, code, description, credits) VALUES (?, ?, ?, ?)`.

  ### 6. Camino de Retorno
  1. El repositorio persiste el registro y retorna el objeto `Course` actualizado con su clave autogenerada.
  2. El servicio mapea la entidad guardada a un `CourseDTO` y lo devuelve al controlador.
  3. El controlador encapsula el DTO en un `ResponseEntity` y retorna el estado `201 Created` al frontend.
  4. La función `handleResponse` en `api.js` valida que la respuesta sea correcta y la deserializa.
  5. En `Courses.jsx`, la promesa se resuelve con éxito en `handleSubmit`, lo que actualiza el estado local de retroalimentación (`setSuccess('Curso creado con éxito')`), cierra el diálogo modal (`setShowModal(false)`) y ejecuta `fetchCourses()`.
  6. `fetchCourses()` llama a `api.courses.getAll()`. Al retornar la lista actualizada, se ejecuta `setCourses(allCourses || [])`, lo que gatilla un re-renderizado de la interfaz mostrando el nuevo curso en la cuadrícula de catálogo.
