# Requerimiento Funcional: RF 03 - Gestión de cursos (Plantilla Educativa)

Este documento sirve como guía educativa paso a paso para comprender el flujo CRUD (Crear, Leer, Actualizar y Eliminar) completo sobre una entidad de catálogo dentro de la arquitectura **React + Spring Boot + PostgreSQL**.

---

## Explicación del Requerimiento (Genérico)
El objetivo de este requerimiento es administrar un catálogo o inventario de registros lógicos independientes (por ejemplo, asignaturas, cursos, categorías o productos).
- Cualquier usuario con los privilegios adecuados puede listar todos los registros existentes, buscar por filtros específicos, ver detalles, agregar nuevos elementos, editar atributos existentes y dar de baja (eliminar lógica o físicamente) registros obsoletos.

---

## Estructura del Flujo de Comunicación (Plantilla educativa)

### 1. Frontend (React)
El componente `[ComponenteReact]` consume la API REST del backend para listar y actualizar la lista de elementos en la interfaz de usuario.

```javascript
// Ejemplo genérico en React ([ComponenteReact].jsx)
import React, { useEffect, useState } from 'react';

export const [ComponenteReact] = () => {
  const [items, setItems] = useState([]);
  const [newItem, setNewItem] = useState({ code: '', name: '', description: '' });

  useEffect(() => {
    fetchItems();
  }, []);

  const fetchItems = async () => {
    const response = await fetch('[EndpointUrl]', {
      headers: { 'Authorization': 'Bearer ' + localStorage.getItem('token') }
    });
    const data = await response.json();
    setItems(data);
  };

  const handleCreate = async (e) => {
    e.preventDefault();
    const response = await fetch('[EndpointUrl]', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': 'Bearer ' + localStorage.getItem('token')
      },
      body: JSON.stringify(newItem)
    });
    if (response.ok) {
      fetchItems(); // Recargar la lista
      setNewItem({ code: '', name: '', description: '' });
    }
  };

  return (
    <div>
      <h2>Catálogo de Cursos / Elementos</h2>
      <form onSubmit={handleCreate}>
        <input placeholder="Código" value={newItem.code} onChange={e => setNewItem({...newItem, code: e.target.value})} />
        <input placeholder="Nombre" value={newItem.name} onChange={e => setNewItem({...newItem, name: e.target.value})} />
        <button type="submit">Agregar</button>
      </form>
      <ul>
        {items.map(item => (
          <li key={item.id}>{item.code} - {item.name}</li>
        ))}
      </ul>
    </div>
  );
};
```

### 2. Red (HTTP Call)
Dependiendo de la acción seleccionada en el frontend, la petición variará en su método HTTP:

- **Operación Listar (GET)**:
  - **Método**: `GET`
  - **URL**: `[EndpointUrl]` (ej. `http://localhost:8080/api/courses`)
  - **Cabeceras**: `Authorization: Bearer [TokenJWT]`
  - **Cuerpo**: *Ninguno*
  
- **Operación Crear (POST)**:
  - **Método**: `POST`
  - **URL**: `[EndpointUrl]`
  - **Cuerpo (Payload JSON)**:
    ```json
    {
      "code": "INF-202",
      "name": "Estructuras de Datos",
      "description": "Conceptos avanzados de programación."
    }
    ```

### 3. Controller (Spring Boot)
El controlador `[ControllerClass]` expone las diferentes rutas REST asociadas al CRUD.

```java
// Ejemplo genérico en Spring Boot ([ControllerClass].java)
package com.example.generic.controller;

import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;
import java.util.List;

@RestController
@RequestMapping("/api/courses")
public class [ControllerClass] {

    private final [ServiceClass] courseService;

    public [ControllerClass]([ServiceClass] courseService) {
        this.courseService = courseService;
    }

    @GetMapping
    public ResponseEntity<List<[CourseDTO]>> getAll() {
        return ResponseEntity.ok(courseService.getAllCourses());
    }

    @PostMapping
    public ResponseEntity<[CourseDTO]> create(@RequestBody [CourseDTO] courseDTO) {
        return ResponseEntity.ok(courseService.createCourse(courseDTO));
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id) {
        courseService.deleteCourse(id);
        return ResponseEntity.noContent().build();
    }
}
```

### 4. Service
El servicio `[ServiceClass]` valida la consistencia del catálogo antes de guardar o modificar (por ejemplo, validando que el código único del elemento no exista ya).

```java
// Ejemplo genérico en Spring Boot ([ServiceClass].java)
package com.example.generic.service;

import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import java.util.List;
import java.util.stream.Collectors;

@Service
public class [ServiceClass] {

    private final [RepositoryInterface] courseRepository;

    public [ServiceClass]([RepositoryInterface] courseRepository) {
        this.courseRepository = courseRepository;
    }

    @Transactional(readOnly = true)
    public List<[CourseDTO]> getAllCourses() {
        return courseRepository.findAll().stream()
                .map(this::convertToDTO)
                .collect(Collectors.toList());
    }

    @Transactional
    public [CourseDTO] createCourse([CourseDTO] dto) {
        if (courseRepository.existsByCode(dto.getCode())) {
            throw new RuntimeException("El código de curso ya está registrado");
        }

        [JPAEntity] entity = new [JPAEntity]();
        entity.setCode(dto.getCode());
        entity.setName(dto.getName());
        entity.setDescription(dto.getDescription());

        [JPAEntity] saved = courseRepository.save(entity);
        return convertToDTO(saved);
    }

    @Transactional
    public void deleteCourse(Long id) {
        if (!courseRepository.existsById(id)) {
            throw new RuntimeException("El curso no existe");
        }
        courseRepository.deleteById(id);
    }

    private [CourseDTO] convertToDTO([JPAEntity] entity) {
        [CourseDTO] dto = new [CourseDTO]();
        dto.setId(entity.getId());
        dto.setCode(entity.getCode());
        dto.setName(entity.getName());
        dto.setDescription(entity.getDescription());
        return dto;
    }
}
```

### 6. Return Path
El flujo de retorno devuelve la respuesta al cliente:

1. **Base de Datos (PostgreSQL)** ejecuta la consulta u operación DML y confirma el éxito al backend.
2. **`[RepositoryInterface]`** retorna la entidad persistida o el listado de registros mapeados desde las filas de la tabla de la base de datos.
3. **`[ServiceClass]`** transforma la lista o entidad de base de datos a DTOs para aislar el modelo físico.
4. **`[ControllerClass]`** serializa los DTOs en un array/objeto JSON y retorna la respuesta HTTP `200 OK` (o `204 No Content` si se eliminó).
5. **Red**: Transporta los datos serializados en JSON.
6. **Frontend (React)**: El código JS resuelve la llamada asíncrona. La lista devuelta actualiza el estado local (`setItems`), forzando un re-renderizado automático en la interfaz de usuario para mostrar la información fresca en pantalla.
