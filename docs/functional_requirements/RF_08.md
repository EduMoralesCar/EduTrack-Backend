# RF 08: Registro de notas

- **Explicación del Requerimiento**:
  Este requerimiento permite a los docentes (`TEACHER`) y administradores (`ADMIN`) registrar y modificar calificaciones individuales obtenidas por los estudiantes en una determinada evaluación. El sistema valida que la nota ingresada esté en el rango de 0 a 20 (escala vigesimal estándar en Perú). Además, cada registro de nota recalculará los promedios ponderados del alumno en el curso, evaluará si entra en estado de riesgo académico (por ejemplo, si el promedio proyectado es desaprobatorio) y enviará una notificación automática en tiempo real al estudiante informando sobre su calificación.

  4. **Capa de Negocio (Backend Service)**:
     - **Nombre de Clase**: [GradeService.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/domain/service/GradeService.java)
     - **Implementación del Método**: `recordGrade`
     - **Mapeo de DTO**: Mapea campos del `GradeDTO` a la entidad `Grade`. Si el registro ya existe, actualiza el puntaje y comentario.
     - **Validaciones**:
       - Valida que la nota esté dentro del rango permitido (`score < 0.0 || score > 20.0`), arrojando `BusinessException("La nota debe estar entre 0 y 20.")`.
       - Valida la existencia del estudiante y evaluación mediante sus repositorios.
       - Dispara procesos secundarios en bloques try-catch: `enrollmentService.evaluateAcademicRisk` para alertar riesgos académicos e `notificationService.sendNotification` para notificar al alumno.
     - **Snippet de Código Java**:
       ```java
       @Transactional
       public GradeDTO recordGrade(GradeDTO dto) {
           if (dto.getScore() < 0.0 || dto.getScore() > 20.0) {
               throw new BusinessException("La nota debe estar entre 0 y 20.");
           }

           Assignment assignment = assignmentRepository.findById(dto.getAssignmentId())
                   .orElseThrow(() -> new ResourceNotFoundException("Assignment not found"));
           User student = userRepository.findById(dto.getStudentId())
                   .orElseThrow(() -> new ResourceNotFoundException("Student not found"));

           List<Grade> grades = gradeRepository.findByAssignmentIdAndStudentId(dto.getAssignmentId(), dto.getStudentId());
           Grade grade;

           if (!grades.isEmpty()) {
               // Si ya existe la nota, se actualiza el score
               grade = grades.get(0);
               grade.setScore(dto.getScore());
               if (dto.getTeacherComment() != null) {
                   grade.setTeacherComment(dto.getTeacherComment());
               }
               // Limpieza de duplicados residuales si existieran
               if (grades.size() > 1) {
                   for (int i = 1; i < grades.size(); i++) {
                       gradeRepository.delete(grades.get(i));
                   }
               }
           } else {
               // Si es nueva, se construye
               grade = Grade.builder()
                       .assignment(assignment)
                       .student(student)
                       .score(dto.getScore())
                       .teacherComment(dto.getTeacherComment())
                       .build();
           }

           Grade saved = gradeRepository.save(grade);
           dto.setId(saved.getId());

           try {
               enrollmentService.evaluateAcademicRisk(dto.getStudentId(), assignment.getSection().getId());
               notificationService.sendNotification(student, "Se ha registrado una calificación de " + dto.getScore() + " en la evaluación '" + assignment.getName() + "'.");
           } catch (Exception e) {
               System.err.println("Warning evaluating academic risk / sending notification: " + e.getMessage());
           }

           return dto;
       }
       ```

  5. **Capa de Persistencia (Repository & Entity)**:
     - **Repositorio**: [GradeRepository.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/persistance/repository/GradeRepository.java)
       - **Métodos JPA Utilizados**: `findByAssignmentIdAndStudentId(Long assignmentId, Long studentId)` para buscar notas previamente ingresadas y `save(Grade grade)` para insertar o actualizar en la BD.
     - **Entidad**: [Grade.java](file:///d:/Workspace/utp/EduTrack/Backend/src/main/java/com/utp/EduTrack/persistance/entity/Grade.java)
       - **Tabla de Base de Datos**: `grades`
       - **Detalles de Registro/Actualización/Consulta**: Se ejecuta un query `SELECT` para evaluar la existencia de una nota con el mismo `assignment_id` y `student_id`. Dependiendo de esto, se realiza un `UPDATE` del campo `score` (y opcionalmente `teacher_comment`) o un `INSERT` en la tabla `grades` con sus llaves foráneas y el valor asignado.

  6. **Camino de Retorno**:
     - El servidor retorna la nota grabada serializada como JSON junto al código HTTP `201 Created` (o `200 OK` si fue actualización indirecta).
     - La respuesta es capturada por el frontend en `Grades.jsx`.
     - Se invoca `showNotification('Calificación registrada correctamente.')` para mostrar un banner de éxito flotante.
     - Inmediatamente se ejecutan llamadas paralelas a `api.grades.getFinalGrades(selectedSection)` y `api.grades.getSectionGrades(selectedSection)` para actualizar los estados reactivos `finalGrades` e `individualGrades` correspondientemente, refrescando al instante tanto la grilla de las 18 semanas como el consolidado de promedios sin necesidad de recargar manualmente la página.
