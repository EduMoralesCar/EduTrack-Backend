# RF 08: Registro de notas

- **Explicación del Requerimiento (Genérico)**:
  Este requerimiento define el flujo para registrar y modificar las calificaciones numéricas obtenidas por los alumnos en evaluaciones específicas. Los docentes pueden visualizar un listado de estudiantes y asignarles una nota. El sistema valida que la calificación ingresada se encuentre dentro de un rango permitido (por ejemplo, 0 a 20) y actualice de forma automática las ponderaciones o promedios acumulados en la base de datos PostgreSQL.

- **Estructura del Flujo de Comunicación (Plantilla educativa)**:

  ### 1. Frontend (React)
  El componente React `[ComponenteReact]` muestra la lista de alumnos junto con casillas de texto para el ingreso de notas. Al hacer clic en guardar o cambiar el valor del campo, se despacha la petición de actualización a la base de datos.

  ```javascript
  // [ComponenteReact].jsx
  import React, { useState } from 'react';

  const [ComponenteReact] = ({ estudianteId, evaluacionId }) => {
    const [nota, setNota] = useState('');

    const handleSaveNota = async () => {
      const notaNum = parseFloat(nota);
      if (isNaN(notaNum) || notaNum < 0 || notaNum > 20) {
        return alert('Nota inválida, debe ser de 0 a 20.');
      }

      try {
        const response = await fetch('[EndpointUrl]', {
          method: 'POST',
          headers: {
            'Content-Type': 'application/json',
            'Authorization': 'Bearer [TokenDeAutenticacion]'
          },
          body: JSON.stringify({ estudianteId, evaluacionId, nota: notaNum })
        });

        if (!response.ok) {
          throw new Error('Error al registrar la calificación');
        }

        const data = await response.json();
        alert('Calificación guardada');
      } catch (err) {
        console.error('Error al guardar nota:', err);
      }
    };

    return (
      <div>
        <input 
          type="number" 
          value={nota} 
          onChange={(e) => setNota(e.target.value)} 
          placeholder="0-20" 
        />
        <button onClick={handleSaveNota}>Guardar Nota</button>
      </div>
    );
  };

  export default [ComponenteReact];
  ```

  ### 2. Red (HTTP Call)
  El cliente envía los datos serializados en JSON mediante la red:
  - **Método HTTP**: `POST` (o `PUT` para actualización)
  - **Ruta de la API**: `[EndpointUrl]` (Ejemplo: `/api/v1/notas`)
  - **Encabezados (Headers)**:
    - `Content-Type: application/json`
    - `Authorization: Bearer [TokenDeAutenticacion]`
  - **Cuerpo del Mensaje (Request Body)**:
    ```json
    {
      "estudianteId": 45,
      "evaluacionId": 12,
      "nota": 18.5
    }
    ```

  ### 3. Controller (Spring Boot)
  La clase controladora `[ControllerClass]` recibe el cuerpo JSON, valida que el usuario remitente sea un docente o administrador autorizado, y delega la ejecución al servicio correspondiente.

  ```java
  // [ControllerClass].java
  package [PackageController];

  import org.springframework.http.HttpStatus;
  import org.springframework.http.ResponseEntity;
  import org.springframework.web.bind.annotation.*;

  @RestController
  @RequestMapping("[EndpointUrl]")
  public class [ControllerClass] {

      private final [ServiceClass] service;

      public [ControllerClass]([ServiceClass] service) {
          this.service = service;
      }

      @PostMapping
      public ResponseEntity<[DTOClass]> registrarNota(@RequestBody [CreateDTOClass] requestDto) {
          [DTOClass] result = service.registrarOActualizarNota(requestDto);
          return ResponseEntity.status(HttpStatus.OK).body(result);
      }
  }
  ```

  ### 4. Service
  La clase `[ServiceClass]` contiene la lógica de negocio. Debe realizar:
  1. Validar que la calificación esté dentro del rango establecido (0 a 20).
  2. Verificar la existencia del estudiante y la evaluación en la base de datos.
  3. Crear o actualizar la entidad de la calificación en la base de datos.
  4. Opcionalmente, recalcular los promedios globales del estudiante en el curso.

  ```java
  // [ServiceClass].java
  package [PackageService];

  import org.springframework.stereotype.Service;
  import org.springframework.transaction.annotation.Transactional;

  @Service
  public class [ServiceClass] {

      private final [RepositoryInterface] repository;

      public [ServiceClass]([RepositoryInterface] repository) {
          this.repository = repository;
      }

      @Transactional
      public [DTOClass] registrarOActualizarNota([CreateDTOClass] dto) {
          // 1. Validar rango
          if (dto.getNota() < 0.0 || dto.getNota() > 20.0) {
              throw new IllegalArgumentException("La nota debe estar entre 0.0 y 20.0");
          }

          // 2. Comprobar si ya existe una nota para esta evaluacion y estudiante
          [JPAEntity] entity = repository
              .findByEstudianteIdAndEvaluacionId(dto.getEstudianteId(), dto.getEvaluacionId())
              .orElse(new [JPAEntity]());

          entity.setEstudianteId(dto.getEstudianteId());
          entity.setEvaluacionId(dto.getEvaluacionId());
          entity.setValorNota(dto.getNota());
          entity.setFechaModificacion(java.time.LocalDateTime.now());

          [JPAEntity] saved = repository.save(entity);

          // Lógica de negocio adicional: Recalcular promedio acumulado
          // [PromedioService].recalcularPromedio(dto.getEstudianteId());

          return mapToDto(saved);
      }

      private [DTOClass] mapToDto([JPAEntity] entity) {
          [DTOClass] dto = new [DTOClass]();
          dto.setId(entity.getId());
          dto.setEstudianteId(entity.getEstudianteId());
          dto.setEvaluacionId(entity.getEvaluacionId());
          dto.setNota(entity.getValorNota());
          dto.setFechaModificacion(entity.getFechaModificacion());
          return dto;
      }
  }
  ```

  ### 5. Repository & Entity
  - **Repository**: La interfaz `[RepositoryInterface]` extiende de `JpaRepository` e introduce métodos para buscar por la combinación única de estudiante y evaluación.
  - **Entity**: La clase `[JPAEntity]` define el mapeo para almacenar la relación entre alumno, evaluación y su calificación en la base de datos PostgreSQL.

  ```java
  // [RepositoryInterface].java
  package [PackageRepository];

  import org.springframework.data.jpa.repository.JpaRepository;
  import org.springframework.stereotype.Repository;
  import java.util.Optional;

  @Repository
  public interface [RepositoryInterface] extends JpaRepository<[JPAEntity], Long> {
      Optional<[JPAEntity]> findByEstudianteIdAndEvaluacionId(Long estudianteId, Long evaluacionId);
  }
  ```

  ### 6. Return Path
  1. **PostgreSQL**: Realiza el comando SQL (insert o update) y retorna el registro almacenado.
  2. **[RepositoryInterface]**: Retorna el objeto `[JPAEntity]` a la capa de servicio.
  3. **[ServiceClass]**: Recibe la entidad guardada, ejecuta cálculos adicionales si corresponde, mapea la entidad al `[DTOClass]` de retorno y lo devuelve al controlador.
  4. **[ControllerClass]**: Transforma la salida del servicio en una respuesta HTTP `200 OK` que contiene el objeto JSON serializado.
  5. **Red**: Transporta el JSON del backend al navegador web.
  6. **[ComponenteReact]**: Recibe la respuesta en la promesa de React, confirmando el registro exitoso en la base de datos, y puede actualizar la fila de la tabla de calificaciones del estudiante de forma reactiva en pantalla.
