# Requerimiento Funcional: RF 12 - Consulta de rendimiento académico (Plantilla Educativa)

## Explicación del Requerimiento (Genérico)
Este requerimiento define el flujo de comunicación estándar para consultar o leer los datos detallados de rendimiento académico de un estudiante mediante su identificador único (ID). El objetivo es que la interfaz web solicite la información histórica y actual del rendimiento escolar, el backend la busque en la base de datos PostgreSQL y la retorne estructurada para su visualización.

---

## Estructura del Flujo de Comunicación (Plantilla educativa)

### 1. Frontend (React)
El flujo se inicia en el componente React `[ComponenteReact]`, el cual extrae el ID de la URL o de las props del componente y realiza la consulta en el ciclo de vida (por ejemplo, con `useEffect`).

```javascript
// [ComponenteReact].jsx
import React, { useEffect, useState } from 'react';
import { useParams } from 'react-router-dom';

export const ComponenteRendimiento = () => {
  const { id } = useParams();
  const [rendimiento, setRendimiento] = useState(null);
  const [cargando, setCargando] = useState(true);

  useEffect(() => {
    const obtenerRendimiento = async () => {
      try {
        const datos = await fetchRendimientoPorId(id);
        setRendimiento(datos);
      } catch (error) {
        console.error('Error al obtener rendimiento académico:', error);
      } finally {
        setCargando(false);
      }
    };
    obtenerRendimiento();
  }, [id]);

  if (cargando) return <div>Cargando rendimiento académico...</div>;
  if (!rendimiento) return <div>No se encontraron registros de rendimiento académico</div>;

  return (
    <div>
      <h1>Rendimiento Académico #{rendimiento.id}</h1>
      <p>Estudiante ID: {rendimiento.estudianteId}</p>
      <p>Promedio General: {rendimiento.promedioGeneral}</p>
    </div>
  );
};
```

### 2. Red (HTTP Call)
La consulta viaja por la red a través de una petición HTTP GET con el ID como parámetro de ruta.

- **Método HTTP:** `GET`
- **Ruta/URL:** `[EndpointUrl]/{id}` (Ejemplo: `/api/v1/rendimiento/123`)
- **Cabeceras (Headers):** `Accept: application/json`
- **Cuerpo (Request Body):** Vacío.

### 3. Controller (Spring Boot)
El controlador `[ControllerClass]` recibe el ID de la ruta (`@PathVariable`) y hace la llamada al servicio de negocio.

```java
// [ControllerClass].java
@RestController
@RequestMapping("/api/v1/rendimiento")
public class ControllerClass {

    private final ServiceClass serviceClass;

    public ControllerClass(ServiceClass serviceClass) {
        this.serviceClass = serviceClass;
    }

    @GetMapping("/{id}")
    public ResponseEntity<JPAEntity> obtenerRendimientoPorId(@PathVariable Long id) {
        JPAEntity rendimiento = serviceClass.obtenerPorId(id);
        return ResponseEntity.ok(rendimiento);
    }
}
```

### 4. Service
El servicio `[ServiceClass]` coordina la búsqueda del registro en la base de datos y gestiona los escenarios en los que el recurso no existe lanzando excepciones personalizadas.

```java
// [ServiceClass].java
@Service
public class ServiceClass {

    private final RepositoryInterface repositoryInterface;

    public ServiceClass(RepositoryInterface repositoryInterface) {
        this.repositoryInterface = repositoryInterface;
    }

    @Transactional(readOnly = true)
    public JPAEntity obtenerPorId(Long id) {
        return repositoryInterface.findById(id)
                .orElseThrow(() -> new ResourceNotFoundException("Registro de rendimiento no encontrado con ID: " + id));
    }
}
```

### 5. Repository & Entity
La capa de persistencia `[RepositoryInterface]` consulta la base de datos PostgreSQL utilizando Spring Data JPA. La entidad `[JPAEntity]` mapea los registros.

```java
// [RepositoryInterface].java
@Repository
public interface RepositoryInterface extends JpaRepository<JPAEntity, Long> {
}

// [JPAEntity].java
@Entity
@Table(name = "tabla_rendimiento")
public class JPAEntity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "estudiante_id", nullable = false)
    private Long estudianteId;

    @Column(name = "promedio_general", nullable = false)
    private Double promedioGeneral;

    // Getters y Setters
}
```

### 6. Return Path
El flujo de retorno es el siguiente:
1. **Base de Datos:** Ejecuta la sentencia SQL `SELECT` y retorna los campos al Driver de conexión JDBC.
2. **Repository:** Mapea el resultado obtenido a una instancia de `JPAEntity` (envuelta en un `Optional`).
3. **Service:** Obtiene la entidad del `Optional` y la devuelve al controlador.
4. **Controller:** Retorna un `ResponseEntity.ok()` con la entidad serializada en JSON.
5. **Red:** Viaja la respuesta HTTP `200 OK` con los datos JSON al cliente.
6. **Frontend (React):** `fetchRendimientoPorId` recibe los datos, actualiza el estado local y renderiza la información académica en pantalla.
