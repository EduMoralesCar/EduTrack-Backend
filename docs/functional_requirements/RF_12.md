# Requerimiento Funcional: [RF_12 - Nombre del Requerimiento de Consulta] (Plantilla Educativa)

## Explicación del Requerimiento (Genérico)
Esta plantilla describe el flujo de comunicación estándar para la **consulta o lectura** de un recurso específico mediante su identificador único (ID). El objetivo es permitir que un usuario navegue a una pantalla de detalles y se carguen sus datos actuales de forma dinámica desde el backend (PostgreSQL).

---

## Estructura del Flujo de Comunicación (Plantilla educativa)

### 1. Frontend (React)
El flujo comienza en el componente React `[ComponenteReactDetalle]`, típicamente al cargar la vista usando un hook como `useEffect` y extrayendo el ID de la URL.

```javascript
// [ComponenteReactDetalle].jsx
import React, { useEffect, useState } from 'react';
import { useParams } from 'react-router-dom';

export const ComponenteDetalle = () => {
  const { id } = useParams();
  const [recurso, setRecurso] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const cargarDetalles = async () => {
      try {
        const data = await fetchRecursoPorId(id);
        setRecurso(data);
      } catch (error) {
        console.error('Error al cargar detalles:', error);
      } finally {
        setLoading(false);
      }
    };
    cargarDetalles();
  }, [id]);

  if (loading) return <div>Cargando...</div>;
  if (!recurso) return <div>Recurso no encontrado</div>;

  return (
    <div>
      <h1>Detalles del Recurso #{recurso.id}</h1>
      <p>Campo Uno: {recurso.campoUno}</p>
      <p>Campo Dos: {recurso.campoDos}</p>
    </div>
  );
};
```

### 2. Red (HTTP Call)
La llamada de red se realiza mediante una petición HTTP GET.

- **Método HTTP:** `GET`
- **Ruta/URL:** `[EndpointUrl]/{id}` (Ejemplo: `/api/v1/recursos/123`)
- **Cabeceras (Headers):** `Accept: application/json`
- **Cuerpo (Request Body):** Vacío.

### 3. Controller (Spring Boot)
El controlador `[ControllerClass]` expone el endpoint GET con la variable de ruta (`@PathVariable`).

```java
// [ControllerClass].java
@RestController
@RequestMapping("/api/v1/recursos")
public class ControllerClass {

    private final ServiceClass serviceClass;

    public ControllerClass(ServiceClass serviceClass) {
        this.serviceClass = serviceClass;
    }

    @GetMapping("/{id}")
    public ResponseEntity<JPAEntity> obtenerPorId(@PathVariable Long id) {
        JPAEntity recurso = serviceClass.obtenerPorId(id);
        return ResponseEntity.ok(recurso);
    }
}
```

### 4. Service
El servicio `[ServiceClass]` busca el recurso y lanza una excepción adecuada si no existe.

```java
// [ServiceClass].java
@Service
public class ServiceClass {

    private final RepositoryInterface repositoryInterface;

    public ServiceClass(RepositoryInterface repositoryInterface) {
        this.repositoryInterface = repositoryInterface;
    }

    @Transactional(readOnly = true)
    public JPAEntity obtenerPorId(Long id) {
        return repositoryInterface.findById(id)
                .orElseThrow(() -> new ResourceNotFoundException("Recurso no encontrado con ID: " + id));
    }
}
```

### 5. Repository & Entity
La interfaz `[RepositoryInterface]` utiliza el método `findById` predefinido de Spring Data JPA.

```java
// [RepositoryInterface].java
@Repository
public interface RepositoryInterface extends JpaRepository<JPAEntity, Long> {
}

// [JPAEntity].java
@Entity
@Table(name = "tabla_recursos")
public class JPAEntity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "campo_uno")
    private String campoUno;

    @Column(name = "campo_dos")
    private String campoDos;

    // Getters y Setters
}
```

### 6. Return Path
El flujo de retorno es el siguiente:
1. **Base de Datos:** Ejecuta la consulta SQL (`SELECT ... WHERE id = ?`) y devuelve el registro al Driver JDBC.
2. **Repository:** Mapea el registro de la BD a una instancia de `JPAEntity` envuelta en un `Optional`.
3. **Service:** Obtiene el valor del `Optional` o lanza excepción (que se manejaría con un `@ControllerAdvice`). Retorna el objeto a `[ControllerClass]`.
4. **Controller:** Retorna un `ResponseEntity` con el estado `200 OK` y el cuerpo JSON.
5. **Red:** Viaja la respuesta HTTP con la representación JSON del recurso.
6. **Frontend (React):** Recibe el JSON, actualiza el estado `recurso` y React vuelve a renderizar la interfaz mostrando los datos cargados.
