# Requerimiento Funcional: [RF_15 - Nombre del Requerimiento de Listado y Paginación] (Plantilla Educativa)

## Explicación del Requerimiento (Genérico)
Esta plantilla describe el flujo de comunicación estándar para la **obtención de listados** paginados, ordenados y filtrados. Esto optimiza el consumo de memoria del servidor e internet enviando únicamente un subconjunto de datos (páginas) y permitiendo al usuario navegar con controles de paginación.

---

## Estructura del Flujo de Comunicación (Plantilla educativa)

### 1. Frontend (React)
El flujo comienza en el componente React `[ComponenteReactPaginado]`. Este maneja variables de estado para la página actual, tamaño de página y parámetros de búsqueda.

```javascript
// [ComponenteReactPaginado].jsx
import React, { useEffect, useState } from 'react';

export const ComponentePaginado = () => {
  const [elementos, setElementos] = useState([]);
  const [paginaActual, setPaginaActual] = useState(0);
  const [totalPaginas, setTotalPaginas] = useState(0);
  const [filtro, setFiltro] = useState('');

  useEffect(() => {
    const cargarPagina = async () => {
      try {
        const response = await fetchListadoPaginado(paginaActual, 10, filtro);
        setElementos(response.content); // Contenido de la página
        setTotalPaginas(response.totalPages); // Páginas totales devueltas por Spring Data
      } catch (error) {
        console.error('Error al listar recursos:', error);
      }
    };
    cargarPagina();
  }, [paginaActual, filtro]);

  return (
    <div>
      <input type="text" placeholder="Buscar..." onChange={(e) => { setFiltro(e.target.value); setPaginaActual(0); }} />
      <ul>
        {elementos.map(item => <li key={item.id}>{item.campoUno}</li>)}
      </ul>
      <button disabled={paginaActual === 0} onClick={() => setPaginaActual(paginaActual - 1)}>Anterior</button>
      <span>Página {paginaActual + 1} de {totalPaginas}</span>
      <button disabled={paginaActual >= totalPaginas - 1} onClick={() => setPaginaActual(paginaActual + 1)}>Siguiente</button>
    </div>
  );
};
```

### 2. Red (HTTP Call)
La llamada de red se realiza usando una petición HTTP GET con parámetros de consulta (`query params`).

- **Método HTTP:** `GET`
- **Ruta/URL:** `[EndpointUrl]?page={pagina}&size={tamanio}&filtro={texto}` (Ejemplo: `/api/v1/recursos?page=0&size=10&filtro=prueba`)
- **Cabeceras (Headers):** `Accept: application/json`
- **Cuerpo (Request Body):** Vacío.

### 3. Controller (Spring Boot)
El controlador `[ControllerClass]` recibe los parámetros opcionales y los convierte en un objeto de paginación (`Pageable`).

```java
// [ControllerClass].java
@RestController
@RequestMapping("/api/v1/recursos")
public class ControllerClass {

    private final ServiceClass serviceClass;

    public ControllerClass(ServiceClass serviceClass) {
        this.serviceClass = serviceClass;
    }

    @GetMapping
    public ResponseEntity<Page<JPAEntity>> listarPaginado(
            @RequestParam(defaultValue = "0") int page,
            @RequestParam(defaultValue = "10") int size,
            @RequestParam(required = false) String filtro) {
        
        Pageable pageable = PageRequest.of(page, size, Sort.by("id").ascending());
        Page<JPAEntity> resultado = serviceClass.listarPaginado(filtro, pageable);
        return ResponseEntity.ok(resultado);
    }
}
```

### 4. Service
El servicio `[ServiceClass]` filtra por las propiedades correspondientes utilizando Spring Data JPA.

```java
// [ServiceClass].java
@Service
public class ServiceClass {

    private final RepositoryInterface repositoryInterface;

    public ServiceClass(RepositoryInterface repositoryInterface) {
        this.repositoryInterface = repositoryInterface;
    }

    @Transactional(readOnly = true)
    public Page<JPAEntity> listarPaginado(String filtro, Pageable pageable) {
        if (filtro != null && !filtro.trim().isEmpty()) {
            return repositoryInterface.findByCampoUnoContainingIgnoreCase(filtro, pageable);
        }
        return repositoryInterface.findAll(pageable);
    }
}
```

### 5. Repository & Entity
La interfaz `[RepositoryInterface]` extiende de `JpaRepository` (que a su vez extiende de `PagingAndSortingRepository`) y declara métodos personalizados de consulta con parámetros de paginación.

```java
// [RepositoryInterface].java
@Repository
public interface RepositoryInterface extends JpaRepository<JPAEntity, Long> {
    Page<JPAEntity> findByCampoUnoContainingIgnoreCase(String filtro, Pageable pageable);
}

// [JPAEntity].java
@Entity
@Table(name = "tabla_recursos")
public class JPAEntity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "campo_uno")
    private String campoUno;
}
```

### 6. Return Path
El flujo de retorno es el siguiente:
1. **Base de Datos:** Ejecuta dos consultas SQL: una para obtener los registros paginados (`SELECT ... LIMIT ? OFFSET ?`) y otra para contar el total de elementos coincidentes (`SELECT COUNT(...)`).
2. **Repository:** Empaqueta los resultados en un objeto de tipo `Page<JPAEntity>`.
3. **Service:** Devuelve el objeto `Page` al controlador.
4. **Controller:** Retorna la respuesta HTTP `200 OK` conteniendo la metadata de paginación en formato JSON (`content`, `totalPages`, `totalElements`, `size`, `number`, etc.).
5. **Red:** Envía la respuesta JSON estructurada al cliente.
6. **Frontend (React):** Extrae la lista de recursos y la metadata para actualizar su UI y deshabilitar o habilitar controles de paginación.
